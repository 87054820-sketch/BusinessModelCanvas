# Pattern Case Review

This review tracks Invent / Shift Pattern candidates from *The Invincible Company* and their relationship to existing PinGarden cases.

## Boundary

- Strategy frameworks explain how to analyze or manage.
- Business-model patterns explain a reusable model structure or shift.
- Invent / Shift Patterns belong in `packages/case-library/patterns/`, not `strategy-frameworks`.

## V1 pattern candidates

| Pattern | Type | Why V1 | Candidate cases |
|---|---|---|---|
| `from-closed-to-open-innovation` | Shift | Strongly supported by existing Open Business Models cases. | `procter-gamble-cd`, `innocentive`, `glaxosmithkline-patent-pool` |
| `from-sales-to-platform` | Shift | Helps explain platform transitions beyond generic innovation. | `nvidia-cuda`, `salesforce-crm`, `alibaba-group` |
| `from-transactional-to-recurring-revenue` | Shift | Helps readers understand subscription / recurring revenue shifts. | `spotify`, `salesforce-crm`, `nespresso` |

## Story support rule

A pattern tag is complete only when the case story explains the mechanism:

- original model or baseline
- new model or pattern
- BMC blocks changed
- trade-offs and failure modes
- transfer lesson

## Deferred candidates

The book has a wider Invent / Shift library. Defer full import until V1 patterns prove useful in stories and validation.
