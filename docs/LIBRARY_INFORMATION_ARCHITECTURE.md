# Strategy Library Information Architecture

## 顶层定义

策略库是 PinGarden 的学习与方法入口。它把真实企业案例、分析画布、商业模式、实验设计、战略框架和资料来源组织成一条路径：从理解问题，到选择方法，再到创建自己的战略项目。

## Tab 层级

1. **企业案例 / Company Cases**
   - 应用层。
   - 展示真实公司、行业或对比分析。
   - 每个案例应连接画布、故事、商业模式或战略框架。

2. **分析画布 / Analysis Canvases**
   - 工作工具层。
   - 展示用户可以直接使用的结构化画布。
   - 画布不是资料附件，而是把问题转化为可编辑工作空间的方法。

3. **商业模式 / Business Models**
   - 模式层。
   - 解释可复用的商业结构，如长尾、多边平台、免费、开放创新。
   - 必须有案例示例支撑。

4. **实验设计 / Experiment Design**
   - 验证层。
   - 帮助用户把画布上的高风险假设转成可证伪实验。
   - 应说明适用风险、成本、证据强度和常用画布。

5. **战略分析 / Strategy Analysis**
   - 视角层。
   - 提供竞争、环境、组合、情景、平台等分析框架。
   - 每个框架都应连接案例与画布。

6. **资料 / Resources**
   - 来源层。
   - 收录书籍、文章、报告、论文和公开资料。
   - 每条资料必须说明推荐理由、关联画布、关联案例或关联方法。

## 关系规则

- 案例回答“这个问题在真实公司里如何出现”。
- 画布回答“我如何开始分析”。
- 模式回答“这类商业结构是什么”。
- 实验回答“我如何验证关键假设”。
- 战略框架回答“我应该从什么视角判断”。
- 资料回答“这些方法和案例从哪里来”。

## 维护规则

- 新增画布时，必须确认它在策略库中如何被解释，并至少关联一个案例或方法。
- 新增案例时，必须明确它支撑哪些模式、框架或画布。
- 新增资料时，不能只登记引用，必须写推荐理由和关联内容。
- 新增 Tab 前先判断是否属于现有六层之一。
