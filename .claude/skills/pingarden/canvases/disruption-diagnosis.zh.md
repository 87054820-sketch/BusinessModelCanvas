# 颠覆诊断画布

## 一句话总结

这是一张**判定**画布，不是「我们很颠覆」的勋章。要么通过 Christensen 严格三段判定，要么诚实地承认这只是维持性创新 / 新市场进入 / 直接竞争——三者都是有效动作，但都不是颠覆。

## 填写顺序

1. **立足点客户**先填。候选既不是被过度满足的主流也不是不消费者——就不是颠覆，立即停止。
2. **更差的维度** + **可接受的维度**成对填。颠覆是一笔交易——写明放弃了什么、换来了什么。
3. **在位者理性忽视理由**。在位者既能又愿意出手——你在竞争，不在颠覆。
4. **向上迁移路径**。具体里程碑，不是「我们会随时间变好」。
5. **最致命假设**最后填——哪个假设破了就崩盘？

## 跨块不变量

- 严格判定（Christensen 2015）：3 条必须同时成立。(1) 立足点在低端或不消费区；(2) 起步在主流维度更差；(3) 持续沿主流维度向上改进且不丢立足点优势。
- 「可接受的维度」必须解释**立足点客户为什么接受那个「更差」**。「便宜」单独通常不够——这个价格解锁了他们原本得不到的什么？
- 迁移路径至少要给 2 个具体的下一步客户群——不是空泛的「以后扩展」。

## 反模式

- 把任何新东西都贴「颠覆性」标签。Uber 案例是教科书级误用。
- 把新技术与颠覆性创新混为一谈。新技术用在主流客户的现有维度上仍然是维持性。
- 跳过立足点判定——没有过度满足或不消费的客户基础，你拿到的就是直接竞争。
- 「向上迁移」写成一厢情愿，没有用客户群证据支撑。
- 把颠覆性押注塞在在位者的标准利润指标下——Christensen 的组织处方是隔离。

## 语气

严格，有证据支撑。区别于蓝海 / 新市场进入的关键是**严格性**——立足点 + 起步劣势 + 向上改进的三件套必须齐全。数字胜过形容词。如果一张 sticky 写「用户喜欢简单」，重写成「用户在 90 秒内完成入门，在位者需要 4 分钟，样本 n=120」。

## Blocks

`zoneId` 是 API 的稳定标识符,`pingarden canvas write` payload 必须用它,不要翻译。

### `foothold-customer` — 立足点客户

**Prompt** — 立足点是谁？被过度满足的主流客户（低端颠覆），还是在位者从未服务的不消费者（新市场颠覆）？写清细分、地理、场合。

**Example** — 非洲功能机用户——从苹果视角看是不消费者

**Quality bar** — 立足点客户是颠覆性创新理论的承重假设。 没有可识别的立足点——被过度满足的主流客户或不消费者——看起来像颠覆的东西通常只是直接竞争或维持性创新。

### `worse-on-dimension` — 更差的维度

**Prompt** — 在主流客户看重的哪些维度上颠覆者**起步更差**？写出维度名 + 差距量化——Christensen 严格判定要求这一条。

**Example** — MP3 起步音质低于 CD

**Quality bar** — 这是"颠覆"标签的**承重**判定。 如果颠覆者在任何主流维度上都不是起步劣势，你拿到的就不是颠覆——而是维持性创新、新市场进入或直接竞争。 三者都是有效动作；都不是 Christensen 意义上的颠覆。 要点是对劣势**诚实**。

### `acceptable-on-dimension` — 可接受的维度

**Prompt** — 颠覆者在哪些方面更好？立足点客户为什么愿意接受上面那个「更差」？价格、简单、可获得、便利还是场景匹配？

**Example** — 价格低 50% + 多日续航（Transsion）

**Quality bar** — 颠覆的交换：立足点客户换来了什么，能补偿上面那个劣势？ "便宜"单独通常不够——这个价格**解锁**了什么客户原本得不到的东西？ 便利、简单、可获得、自主、场景匹配、更低承诺、更少前置条件？

### `incumbent-rational-ignore` — 在位者理性忽视理由

**Prompt** — 在位者为什么**理性地**不会出手？典型原因：利润薄、当下市场小、自家高利润产品被蚕食的风险。他们的客户访谈正确地告诉他们应当忽略你。

**Example** — 苹果无法在不毁掉品牌-利润模型的前提下做 50 美元以下手机

**Quality bar** — 这是 Christensen 理论的分析支点：在位者并不蠢，他们**理性**。 他们的资源配置流程正确地说"忽略这个立足点"——基于利润率、市场规模和蚕食风险。 颠覆之所以成功，正是因为在位者的客户访谈**正确地**告诉它要看别处。

### `upmarket-migration-path` — 向上迁移路径

**Prompt** — 具体轨迹：你打算如何在保留立足点优势的同时，沿主流维度改进？产品里程碑、客户群迁移、规模经济。

**Example** — 功能机 → Android 智能机进入新兴市场主流

**Quality bar** — 颠覆故事如果没有轨迹就是不完整的。 立足点本身只是一个利基；让它成为颠覆的是进入者**随时间向高端迁移**——沿主流维度改进，同时保留立足点侧优势。 等在位者注意到时，颠覆者已穿过主流"够用线"并握住客户关系。

### `riskiest-assumptions` — 最致命假设

**Prompt** — 哪个假设破了就崩盘？从下列里挑：立足点客户意愿、改进轨迹可行性、在位者继续忽视的时长、与母公司利润指标的组织隔离能力。

**Example** — ≥X% 的立足点客户真的会以我们承诺的价格买单

**Quality bar** — 那条假设，如果不成立就会证伪颠覆假设并杀掉押注。 通常有 3-4 个候选；挑出最致命的一个，把第一个实验设计在它周围。 Christensen 的组织处方（独立单元、独立指标、耐心资本）只有在这些假设被早期测试时才有意义。

## Colour legend

- `0` — **在位者基线**: 在位者目前如何沿传统维度服务主流客户。
- `1` — **颠覆者立足点**: 新进入者起点：被过度满足的主流客户，或在位者理性忽视的不消费者。
- `2` — **迁移证据**: 沿主流维度改进轨迹的证据：产品里程碑、客户群迁移、规模曲线。
- `3` — **最致命假设**: 立足点意愿、改进轨迹或在位者继续忽视的时长——哪一条最可能先破。
- `4` — **严格判定备注**: 对照 Christensen 三段判定（低端或不消费立足点 / 起步主流维度更差 / 沿主流维度向上改进）的备注——这个候选真的够格吗？

## Pairs with

这张画布常和这些一起用,做完后引导用户接下去做:

- `business-model-canvas` — 颠覆几乎从来不只是产品故事。并排画在位者 BMC 和颠覆者 BMC，指出在哪里分叉——渠道、成本结构、关键活动、伙伴。
- `portfolio-map` — 颠覆性押注深处于 Explore 象限，高风险、收益不确定。立足点 → 主流 → 高端的轨迹在 Portfolio Map 上变成一条 pin 迁移路径。
- `value-proposition-canvas` — 新市场颠覆的客户侧是不消费者——从未被服务过的痛和工作。低端颠覆的客户侧是被过度满足的主流客户——来自在位者「过度工程」的痛。
- `three-horizons-map` — 颠覆候选通常是 H3 押注。先用 Disruption Diagnosis 通过 Christensen 严格判定，再放进三层增长做时间组合排序，再用创新指标度量。
