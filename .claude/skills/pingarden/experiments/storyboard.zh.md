# 故事板

> 6-12 格漫画式序列,带目标客户走过 "你的 idea 出现在他们生活中之前、之中、之后" 的时刻。通过他们的反应做 Discovery。

## Slug

`storyboard` — referenced by AI agents matching a riskiest assumption to a candidate experiment. Cross-reference into canvases is via `experiment.appliesToCanvases[]`.

# 故事板 — AI Agent 速查

**什么时候推荐。** 价值主张是情境化的("背景 X"、"后台同步"),bullets 说不清。或不同客户可能看到不同价值路径、团队需要 commit。或在动工之前让干系人对"体验是什么"对齐。

**证据强度。** 弱。发现反应和缺失痛点;**不验证** 付费意愿。

**样本。** N=5-8 目标客户,一次一个。

**准备 / 执行。** 小时(草图)/ 小时(会话)。

**能力。** `narrative-design`、`sketching`、`interview-design`。

**决策树。**

- 已经有可点击 prototype? → 改推 Clickable Prototype —— 同等成本但保真度更高。
- 方案导向的团队会过度精修? → 把这点点出来;坚持火柴人。
- 用户要验证购买意愿? → 不推 Storyboard。推 Pre-Sale 或 Smoke Test。

**反模式。**

- 漂亮插画暗示已经精修,反馈会拘谨。
- 一格最多一句话。
- "我超喜欢" = 附和,不是验证。

**跨画布。** 产出喂给 `value-proposition-canvas`(通过反应浮出的 jobs/pains/gains)、`customer-journey`(故事板本身就是 journey 的草稿)、`jobs-to-be-done`(一天里的 moments of need)。
