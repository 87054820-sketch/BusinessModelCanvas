# 回旋镖测试 — AI Agent 速查

**什么时候推荐。** 直接或近似竞品存在,用户想在动手造之前发现既有方案的真实摩擦。或者只靠访谈浮不出具体的使用卡点。

**证据强度。** 中偏弱。发现切入点;**不验证** 你能赢下这些切入点。

**样本。** N=5-8,目前都没在用竞品。

**准备 / 执行。** 小时(招募 + 任务设计)/ 天(会话 + debrief)。

**能力。** `interview-design`、`structured-observation`。

**决策树。**

- 没有竞品? → 不推荐。改 Storyboard / Clickable Prototype。
- 招的人已经在用竞品? → 取消资格;他们已经适应了,你看不到摩擦。重新招。
- 差异化是全新模态? → Boomerang 测错维度了。改 Wizard of Oz。
- 用户想下"我们超过竞品"的结论? → 把它顶回去 —— Boomerang 发现切入点,不验证赢下。建议 Smoke Test 或 A/B 跟进。

**反模式。**

- 自己演示竞品 ≠ 观察用户。
- "还行" = 附和,不是数据。
- 招已经在用竞品的用户。

**跨画布。** 锐化 `value-proposition-canvas` 痛点缓解(你的差异化)、`customer-journey` 步骤级情感曲线、`empathy-map` Does / Sees 两列。
