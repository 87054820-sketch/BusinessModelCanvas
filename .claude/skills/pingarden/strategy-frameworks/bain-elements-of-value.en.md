# Bain Elements of Value

## Positioning

This is a `customer-value-lens` framework inside the strategy framework library. Its analysis object is the customer-perceived value layers a proposition stands on; use it to deepen `value-proposition-canvas`, `jobs-to-be-done`, `empathy-map`, and `customer-journey` work by naming which of the 30 elements actually carry the value.

## Select this framework when

- The value proposition is vague or overuses words like convenient, premium, simple, or delightful.
- The user already has VPC/JTBD/journey content, but the `gains`, `gain-creators`, or experience benefits are not clearly classified.
- The user wants to explain why a case feels valuable to customers, not merely list features.

## Recommended chain

1. Start with `jobs-to-be-done` to write situation, motivation, functional outcome, and emotional/social jobs.
2. Use `value-proposition-canvas` to carry those `jobs`, `pains`, and `gains` into `pain-relievers` and `gain-creators`.
3. Use Bain to tag the few important gains: `[functional-saves time]`, `[emotional-reduces anxiety]`, `[social-badge value]`, `[higher-order-self-actualization]`.
4. Use `customer-journey` to locate where those values appear, strengthen, or break.
5. If a value element lacks customer evidence, move to `experiment-canvas` or interviews.

## Use these canvases

1. `value-proposition-canvas` — the primary working surface; map selected value elements into `gains`, `gain-creators`, and `pain-relievers`.
2. `jobs-to-be-done` — explain the situation and job behind the value, especially emotional/social jobs.
3. `empathy-map` — test emotional and social value from customer evidence rather than team projection.
4. `customer-journey` — locate where value appears, strengthens, breaks, or is wasted.

## Questions to ask

- Which segment and situation are we analyzing?
- Which functional values are proven?
- Which emotional, social, or higher-level values are evidenced?
- Which few elements should define the proposition, instead of claiming all 30?
- Which VPC `gain`, `gain-creator`, or `pain-reliever` does each value element map to?
- At which journey stage does the customer actually perceive it?

## Red flags

Do not claim all 30 elements. Do not treat elements as marketing copy. Do not add higher-level value before functional value works. Do not stop at a standalone Bain checklist; reconnect the tags to VPC/JTBD/journey work.
