# Performance-Based Scenario Planning

## Select this framework when

- The user needs to run a scenario-planning project, not only draw scenarios.
- Sponsor support, stakeholder participation, implementation, or assessment is a risk.
- The user asks how scenario planning should affect decisions and learning.

## Fill / facilitation order

1. Define purpose, sponsor, scope, participants, and success criteria.
2. Use `business-model-environment` to collect signals and uncertainty candidates.
3. Use `scenario-matrix` to build scenarios and robust moves.
4. Wind-tunnel BMCs, portfolio bets, and design criteria through the scenarios.
5. Decide what changes now, what options to preserve, and what signals to monitor.
6. Assess results: decision impact, learning, perception shifts, and follow-up actions.

## Red flags

- Workshop scheduled before purpose is clear.
- Scenarios are written but never used.
- No assessment plan exists.
- Stakeholders are informed but not involved.
