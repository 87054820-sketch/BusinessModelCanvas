# 贝恩价值元素

贝恩价值元素把客户感知价值拆成 30 个元素，分布在功能、情感、改变人生和社会影响四个层级。在 PinGarden 中，它被归类为 **Customer Value Lens / 客户价值透镜**：仍属于战略分析方法库，但角色是辅助解释价值主张，而不是独立主画布。

## 方法定位

它回答的问题不是“公司应该选择哪条战略路线”，而是：**某个客户细分为什么会感知到价值？这些收益到底属于功能、情感、社会还是更高阶意义？**

因此它适合放在 VPC、JTBD、共情图和客户旅程旁边，作为价值诊断与标注层。它不替代这些画布，也不应该变成“30 个元素全选”的 checklist。

## 何时使用

当团队已经有客户细分，但价值主张仍然很空泛，例如“方便”“高端”“简单”“令人愉悦”时使用。这个框架可以帮助你明确客户到底重视哪些价值元素，以及哪些元素应该被提升、组合或验证。

不要用它声称自己提供所有价值元素。强价值主张通常是围绕某个细分人群，选择少数真正重要的元素打透。

## 如何映射到 PinGarden 画布

- `value-proposition-canvas` 是主工作面：把价值元素标注到 `gains`、`gain-creators` 和 `pain-relievers`，例如 `[功能-节省时间]`、`[情感-降低焦虑]`、`[社会-身份认同]`。
- `jobs-to-be-done` 是上游输入：用情境、动机和结果解释为什么某个价值元素在此刻重要，尤其把情感/社交任务映射到更高阶价值。
- `empathy-map` 是证据检查：从客户自己看到、说出、做出和感受到的事实，验证情感与社会价值是否存在。
- `customer-journey` 是时序定位：找到价值在哪个触点被创造、增强、丢失或断裂。

## 推荐链路

`JTBD` → `Value Proposition Canvas` → `Bain Elements of Value` 价值标注 → `Customer Journey` → `Experiment Canvas`

这条链路的含义是：先确认客户在什么情境下想取得什么进展，再设计痛点缓解和收益创造，然后用 Bain 拆解收益层级，最后在旅程与实验中验证客户是否真的识别并愿意为这些价值付费。

## 使用流程

1. 选择一个客户细分和一个使用情境。
2. 用 JTBD 写清楚功能任务、情感任务和社交任务。
3. 在 VPC 里先写客户 `jobs`、`pains`、`gains`，再写左侧价值地图。
4. 从 VPC 的 `gains` 和 `gain-creators` 中挑出少数关键收益，标注对应的 Bain 价值元素。
5. 只有在有证据时，才加入情感、改变人生或社会影响元素。
6. 用客户旅程定位每个价值元素出现、增强或断裂的阶段。
7. 用实验或真实行为验证客户是否识别这些价值，并愿意为它们付费。

## 如何解读结果

- **功能价值占主导**：价值主张多半在效率、便利、风险、质量、节省时间/成本上竞争，适合先验证基本可用性和替代方案优势。
- **情感/社会价值清晰**：价值主张可能具备品牌、体验、身份、信心或归属感优势，但必须用客户证据支撑。
- **高阶价值太多**：通常说明团队在写愿景或营销口号，而不是客户已感知的价值。
- **VPC 收益很多但 Bain 元素很散**：价值主张可能失焦，需要回到一个核心客户任务重新收敛。

## 示例中看什么

- `nespresso` 组合了质量、感官吸引力、减少努力、身份象征和持续获取。
- `drybar` 围绕体验、减少努力和自信感缩窄服务。
- `stitch-fix` 组合了多样性、个性化、减少努力和信心。
- `citizenm-hotels` 聚焦简化旅行、设计和便捷获取。
- `novo-nordisk-novopen` 在医疗情境中体现功能易用和减轻焦虑。

## 常见误用

- 选择过多元素，导致价值主张失焦。
- 把价值元素当成营销说法，却没有客户证据。
- 忽略客户在什么情境下重视该元素。
- 把功能便利误当成情感安心。
- 在基础功能价值尚未成立前，就声称提供高层级价值。
- 单独画一张“Bain 价值元素画布”，但没有回连 VPC、JTBD 或客户旅程。
