# BCG 增长份额矩阵

BCG 增长份额矩阵是一套经典组合管理框架，用**市场增长率**和**相对市场份额**对业务分类。四类业务 —— 明星、现金牛、问题和瘦狗 —— 帮助管理层决定投入、收割、选择、剥离或重新定位。

## 何时使用

当公司有多个业务、品牌、区域或产品线，而战略问题是**在市场增长和竞争地位假设下如何配置资源**时使用。

如果没有份额数据、组合主要是早期创新选项，或利润压力主要来自行业结构而不是组合位置，就不要单独使用它。此时应搭配 `business-model-environment`、`porters-five-forces` 或 `portfolio-map`。

## 如何映射到 PinGarden 画布

- `bcg-growth-share-matrix` 是可操作的 2×2 画布，每个业务单元作为可讨论的便签/决策对象放入象限。
- `portfolio-map` 是更宽的总入口，当风险、收益、创新/颠覆移动比份额更重要时使用。
- `business-model-canvas` 用来展开任何需要投入、收割、剥离或重新定位的业务。
- `business-model-environment` 用来验证市场增长假设和市场成熟度信号。

## 使用流程

1. 定义可比较单元：业务、产品线、品牌或区域。
2. 用品类或细分市场数据估算每个单元的市场增长率。
3. 相对最大竞争者或相关领导者估算相对市场份额。
4. 把每个单元放入明星、现金牛、问题或瘦狗象限。
5. 添加证据便签，解释增长/份额假设。
6. 选择动作：投入、收割、选择、剥离、重新定位或重新核对数据。
7. 高风险决策在投入前展开为 BMC 和环境分析。

## 如何解读结果

- **明星业务** 需要投入以防守高增长市场中的领导地位。
- **现金牛业务** 在增长较慢的市场中产生现金，可资助经过选择的未来押注。
- **问题业务** 需要选择性投入，只有一部分应成为明星。
- **瘦狗业务** 必须做决定：剥离、收割、重新定位，或因明确战略理由保留。

## 示例中看什么

- `nestle-portfolio` 是最强示例，因为成熟品类和组合动作让现金牛逻辑很清楚。
- `alibaba-group` 展示商业、物流、云和平台业务在增长市场和领导地位上的差异。
- `ping-an-group` 可被理解为金融现金牛与更高增长技术/平台押注的组合。
- `procter-gamble-cd` 展示组合资金和外部创新选择为什么需要清晰资源逻辑。

## 常见误用

- 没有市场增长率或相对份额证据就画矩阵。
- 把象限标签当作自动决策。
- 因为市场增长就给所有问题业务平均投钱。
- 过度收割现金牛，导致组合失去供血引擎。
- 因为内部政治或团队情感保留瘦狗业务。
- 忽略利润率、防御性、协同和商业模式适配。
