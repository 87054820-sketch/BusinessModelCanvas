# Innovation Metrics

## When to use

Use when the user needs to decide whether an Explore project deserves more funding, a pivot, transfer, or retirement. Pair with Portfolio Map, Experiment Canvas, and Evidence Scorecard.

## Workflow

1. Identify the project stage: Explore or Exploit.
2. Name 3–5 critical assumptions.
3. For each assumption, record the experiment, evidence strength, and latest learning.
4. Judge whether risk is decreasing, not whether activity is increasing.
5. Recommend a portfolio action: persevere, pivot, run another test, transfer, or retire.

## Anti-patterns

Do not judge Explore work too early with mature-business KPIs. Do not treat interview counts, meetings, or prototypes as outcomes by themselves.
