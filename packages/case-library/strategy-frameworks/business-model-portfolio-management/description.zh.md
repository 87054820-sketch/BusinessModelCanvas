# 商业模式组合管理

商业模式组合管理帮助一家公司同时做好两件事：开发现有商业模式，探索未来商业模式。它不是为了展示“我们很创新”，而是把整个业务组合看清楚，比较风险与收益，并决定哪些业务要增长、保护、改善、转入、剥离或终止。

## 核心思想

接近“不可战胜”的公司，会在过时之前持续重塑自己。它们一边改善和增长当前业务，一边寻找未来增长引擎。这两个组合的逻辑不同：

- **Explore / 探索组合**：创新项目、新商业模式、新价值主张、新产品和服务。用 **预期收益** 与 **创新风险** 来定位。
- **Exploit / 开发组合**：现有业务、现有价值主张、产品和服务。用 **收益** 与 **死亡 / 颠覆风险** 来定位。

Portfolio Map 把讨论从单个项目的主观意见，变成一张共享的组合视图。

## 管理节奏

使用这个框架时，按三个动作推进：

1. **Visualize / 可视化**：列出同一颗粒度的现有业务和未来选项，把每个对象放到业务组合地图上。
2. **Analyze / 分析**：判断组合是否过度暴露在颠覆风险下，是否对未来投入不足，是否保留了太多低收益 / 高风险下注。
3. **Manage / 管理**：选择组合动作，并持续观察对象如何随时间移动。

## Explore 动作

Explore 动作用于塑造和测试新商业想法，目标是提高预期收益、降低创新风险。

- **Ideate / 构想**：把机会、技术或洞察变成第一版商业模式原型。
- **Persevere / 坚持**：在证据鼓励时继续测试。
- **Pivot / 转向**：当证据显示原路径不可行时，改变商业模式。
- **Retire / 终止**：当证据或战略匹配度不足时，杀掉探索项目。
- **Spinout / 拆出**：把有潜力但不适合内部承接的想法移到公司外部。
- **Transfer / 转入**：把已经充分降风险的模型从 Explore 转入 Exploit。
- **Invest / 投资**：投资外部创业公司或探索项目，强化组合。

## Exploit 动作

Exploit 动作用于管理现有商业模式，目标是提高收益、降低死亡和颠覆风险。

- **Acquire / 收购**：买入外部业务，新增或强化组合位置。
- **Merge / 合并**：把收购或已有业务与现有业务整合。
- **Partner / 伙伴合作**：通过外部伙伴强化商业模式。
- **Invest / 投资**：参股外部业务，补强组合。
- **Improve / 改善**：把过时模式翻新成更有竞争力的模式。
- **Divest / 剥离**：出售或拆出不再适配的业务。
- **Dismantle / 拆除**：结束并解散某个业务。

## 与画布的关系

- 用 **Portfolio Map / 业务组合地图** 做主视图。
- 对任何值得深入的 pin，展开成 **Business Model Canvas / 商业模式画布**。
- 对 Explore 侧的高风险假设，用 **Experiment Canvas / 实验画布** 设计证据。

## 质量标准

好的组合分析必须说清楚分析颗粒度，区分 Explore 与 Exploit，解释每个对象的位置，并提出组合动作。差的分析只是说一家公司“很创新”，却没有展示任何组合管理逻辑。
