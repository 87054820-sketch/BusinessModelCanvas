# 麦肯锡 7S 框架

麦肯锡 7S 是一套组织一致性分析框架，用来检查 **战略、结构、系统、共同价值观、风格、人员和技能** 是否相互支撑。它有价值，是因为战略失败往往不只是战略表述错误，而是组织无法执行。

## 何时使用

当公司已有战略，但运营模式、文化、人才或能力似乎不匹配时使用。它特别适合转型、并购整合、创新治理，以及把新商业模式扩张进核心组织的场景。

不要把它当作泛泛的文化问卷。如果问题是客户价值，用 VPC 或贝恩价值元素；如果问题是组合时间接续，用三层增长。

## 如何映射到 PinGarden 画布

- `innovation-culture-map` 记录结果、行为、促进因素和阻碍因素，体现 7S 的不一致。
- `design-criteria-canvas` 把组织一致性要求转成运营模式设计准则。
- `business-model-canvas` 显示组织必须支撑什么：价值主张、渠道、活动、资源、伙伴、收入和成本。

## 使用流程

1. 从战略开始：组织必须执行什么选择？
2. 映射当前结构和系统如何支持或阻碍这个选择。
3. 映射人员和技能缺口。
4. 识别共同价值观和领导风格如何强化或违背战略。
5. 把缺口转成 `innovation-culture-map` 上的行为和机制便签。
6. 写成运营模式变更的设计准则。
7. 回查当前组织是否真的能交付这张 BMC。

## 示例中看什么

- `bosch-accelerator` 展示保护探索所需的组织机制。
- `procter-gamble-cd` 展示开放式创新只有在系统和能力支撑下才有效。
- `ping-an-group` 展示集团级战略需要金融和科技单元之间的结构、技能和系统配合。
- `patagonia` 展示价值观和运营选择如何强化战略位置。

## 常见误用

- 只是列出七个词，而不诊断它们之间的互动。
- 把共同价值观当口号，而不是可观察的决策方式。
- 因为文化更容易讨论，就忽略系统、激励和治理。
- 等战略已固定后才用 7S，而不是用它测试战略可执行性。
- 没有把组织问题连接回具体商业模式。
