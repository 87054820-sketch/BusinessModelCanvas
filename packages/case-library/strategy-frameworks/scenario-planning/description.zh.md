# 情景规划

情景规划把商业模式环境扫描向前推进一步：不是只列外部机会和威胁，而是围绕关键不确定性构造多个可信未来，并比较战略选择在不同未来下是否仍然成立。

## 核心思想

单一预测容易让团队过早押注一个未来。情景规划要求团队先识别驱动力，再找出真正会改变战略选择的不确定性，组合成若干个足够不同、但都合理的情景。

## 与画布的关系

- 用 `business-model-environment` 收集趋势、市场、行业和宏观信号；
- 用 `scenario-matrix` 把两个关键不确定性转成四个情景；
- 用 `business-model-canvas` 检查每个情景对现有模型的压力；
- 用 `portfolio-map` 比较哪些下注适合多个未来；
- 用 `design-criteria-canvas` 把稳健动作转成下一版设计标准。

## 质量标准

好的情景不是“乐观/中性/悲观”三档预测，而是能迫使团队看到不同战略逻辑的多个世界。结论应该落到：保留哪些选项、提前验证哪些信号、哪些动作现在就值得做。
