# Performance-Based Scenario Planning

Performance-Based Scenario Planning is the organizational execution layer of scenario planning. It does not replace the scenario matrix. It explains how an organization prepares, runs, deploys, and assesses a scenario project so that scenarios affect decisions, mental models, and performance.

## When to use

Use this framework when the user is responsible for a scenario-planning project, not just a scenario workshop. It is especially useful when leadership support, stakeholder participation, project scope, implementation, or assessment will determine whether the work matters.

Do not use it when the user only needs a quick 2×2 scenario exercise. Use `scenario-planning` and `scenario-matrix` for the core futures logic; use this framework when organizational adoption and decision impact matter.

## Five-phase workflow

1. **Project preparation** — clarify purpose, sponsor support, scope, participants, issue focus, and success criteria.
2. **Scenario exploration** — collect signals, trends, stakeholder perspectives, drivers, and uncertainty candidates.
3. **Scenario development** — build internally consistent scenarios that challenge assumptions and mental models.
4. **Scenario implementation** — use scenarios for strategy dialogue, wind-tunneling, BMC pressure tests, portfolio choices, and action design.
5. **Project assessment** — document results, learning, perception shifts, decision impact, and signals to monitor next.

## Canvas mapping

- `business-model-environment` supplies signals and external forces.
- `scenario-matrix` creates the scenario stories and robust moves.
- `business-model-canvas` is stress-tested in each future.
- `portfolio-map` compares which bets remain useful across futures.
- `design-criteria-canvas` captures robust moves and decision rules.

## How it differs from Scenario Planning

`scenario-planning` is the umbrella method for thinking across futures. `performance-based-scenario-planning` is the project system that makes the work usable inside an organization.

## Common misuses

- Starting with a scenario workshop before defining purpose and sponsorship.
- Treating scenario development as the end of the project.
- Failing to deploy scenarios into strategic decisions or portfolio choices.
- Ignoring perception, bias, and mental models.
- Skipping assessment, so the organization cannot learn whether the project helped.
