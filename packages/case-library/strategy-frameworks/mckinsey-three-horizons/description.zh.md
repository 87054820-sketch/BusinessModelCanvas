# 麦肯锡三层增长

麦肯锡三层增长是一套增长组合框架，用来同时管理现在和未来。它把业务分为 **H1 当前核心业务**、**H2 新增长引擎** 和 **H3 未来选项**，帮助管理层判断公司是否过度优化当下，或没有为下一代增长留出足够空间。

## 何时使用

当决策问题是**增长接续**时使用：核心业务应该投入多少，哪些新业务值得扩张，哪些未来选项还应该继续学习。它尤其适合公司级创新组合、集团战略，以及正在面对 S 曲线更新的企业。

不要用它设计单一商业模式、按市场份额分类业务，或排序功能路线图。单一模型用 BMC；市场增长/份额分类用 BCG 增长份额矩阵；风险/收益位置用业务组合地图。

## 如何映射到 PinGarden 画布

- `three-horizons-map` 是操作画布。H1/H2/H3 是可填写区域，迁移动作和证据风险让项目移动变得明确。
- `portfolio-map` 用风险和收益观察同一批项目，适合看 Explore/Exploit 位置。
- `business-model-canvas` 用来把重要层级项目展开为完整商业模式。
- `experiment-canvas` 把 H2/H3 的假设转成实验。
- `evidence-scorecard` 判断某个项目是否有足够证据继续推进、追加投入、转向、暂停或终止。

## 使用流程

1. 定义组合单元：业务线、产品、平台、区域或创新项目。
2. 放入 H1：今天创造现金、利润、数据、渠道或能力的业务。
3. 放入 H2：已有早期证据但尚未成为可靠核心业务的新增长项。
4. 放入 H3：主要仍是假设和学习型押注的未来选项。
5. 对每个 H2/H3 项目写清进入下一层级所需的证据。
6. 写迁移动作：投入、扩张、转入、防守、转向、暂停或终止。
7. 把未解决假设导入实验画布和证据评分卡，再做资金决策。

## 如何解读结果

健康的地图应有足够强的 H1 为公司供血，有可信的 H2 形成下一代增长，也有足够 H3 保持未来更新。最重要的信号是**移动**：H3 要么学习并靠近 H2，要么被终止；H2 要么扩张靠近 H1，要么改变方向。

## 示例中看什么

- `ping-an-group` 展示金融服务核心如何资助平台和技术选项。
- `bosch-accelerator` 展示 H3/H2 学习侧：大量想法进入，用证据筛选，少数继续推进。
- `alibaba-group` 展示围绕商业、物流、云和平台基础设施的多增长引擎。
- `procter-gamble-cd` 展示外部创新如何与内部扩张能力连接。

## 常见误用

- 把 H1/H2/H3 当作固定年份，而不是成熟度层级。
- 用流行技术填满 H3，却没有客户或商业模式假设。
- 把没有可复用证据的项目叫作 H2。
- 让 H1 指标吃掉所有资金，饿死 H2/H3。
- 忘记创新团队到运营团队的组织交接。
- 只画一次地图，从不跟踪移动。
