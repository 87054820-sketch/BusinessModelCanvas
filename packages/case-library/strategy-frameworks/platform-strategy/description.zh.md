# 平台战略

平台战略用于分析多边生态系统，而不是单一企业把产品卖给客户的线性业务。它关注参与方之间的核心交互、网络效应、治理、信任、开放性和变现。

## 核心思想

平台的价值不只来自平台方自己生产什么，而来自它如何让不同参与方互相创造、匹配和消费价值。好的平台分析必须说明：

- 平台连接哪些边；
- 核心交互是什么；
- 价值单元如何产生和匹配；
- 网络效应如何出现；
- 治理规则如何提升质量和信任；
- 变现方式是否破坏参与方激励。

## 与画布的关系

- 用 `platform-ecosystem-map` 做平台机制主视图；
- 用 `business-model-canvas` 解释平台方如何获利；
- 用 `value-proposition-canvas` 分别分析每一边的任务、痛点和收益；
- 用 `customer-journey` 检查核心交互的体验断点；
- 用 `portfolio-map` 比较平台生态中的业务/服务组合。

## 质量标准

好的平台战略分析不是说“有网络效应”，而是说明网络效应从哪里来、为什么会增强、何时会反噬，以及平台治理如何影响增长和信任。
