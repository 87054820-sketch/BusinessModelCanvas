# PESTEL 宏观环境分析

## 何时选择本框架

- 用户在做新市场进入、周期性环境复盘，或为情景规划做前置准备。
- 团队在反复争论「未来」却没人能锚定——PESTEL 把讨论拉回可观察的信号。
- 用户在做五力 / 环境扫描 / BMC 压力测试之前，缺一层宏观上下文。

## 使用这些画布

1. `business-model-environment` — 把每条 PESTEL 信号落到趋势 / 市场 / 行业 / 宏观 4 个分区。
2. `scenario-matrix` — 把影响和不确定性最高的 2 条 PESTEL 力量当情景矩阵的两个轴。
3. `business-model-canvas` — 把每条信号翻译成 BMC 某个块的压力。

## 必问的问题

- 分析单元是什么（行业 + 地理 + 时间窗）？
- 6 类力量分别有哪 3–5 条可观察的信号？（事实，不是形容词。）
- 哪些信号同时是高影响 + 高不确定性？
- 每条高影响信号的一句 `so-what` 是什么，对应到 BMC 哪个块？

## 与其它框架的接力

- **PESTEL → BMEScan** 是承重的管道。PESTEL 产出 20–40 条宏观信号；BMEScan 把其中影响和不确定性最高的 8–12 条吸收到 BMG 4 区（趋势区吸收 PESTEL 的 S/T/E-环境；宏观区吸收 PESTEL 的 E-经济/P/L）。PESTEL 不覆盖、BMEScan 覆盖的：自家行业里的直接竞争对手和利益相关方——所以有 BMC 时一定要收口到 BMEScan。
- 用环境扫描把最贴 BMC 的信号压缩成 BMG 的 4 力。
- 当 2 条以上 PESTEL 力量是高不确定性 + 高影响时，启动情景规划。
- 五力只在 PESTEL 明确宏观上下文之后用——它们是不同高度的工具。

## 红线

不要让 PESTEL 沦为 PPT 单页。不要因为技术故事性强就过度偏重技术。不要把不确定性（要分情景应对）和风险（要打折扣）混为一谈。不要跳过 `so-what`——没有 BMC 蕴含的 PESTEL 格子就是噪声。
