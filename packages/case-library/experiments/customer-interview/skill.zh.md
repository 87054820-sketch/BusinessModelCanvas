# 客户访谈 — AI Agent 速查

**什么时候推荐。** 用户有一个需求性假设但零数据;或正在起 VPC / Empathy Map / JTBD,客户那一侧还是空的;或刚停在某个 "致命假设" 便签上问 "怎么验证这个?"。

**证据强度。** 弱。它探的是问题的框,**不验证** 付费意愿或购买意向。如果用户追问"算不算已经验证了?" —— 直接答否,接着推荐 Smoke Test / Concierge / Letter of Intent。

**样本量门槛。** 下限 N=5,甜蜜点 N=8-12。低于 5 不要谈规律;低于 12 不要说"覆盖了这个细分"。某个主题在 5 之 3 不被提示就独立浮现 —— 通过初筛,可以进一步验证。

**准备时间。** 小时级:讨论提纲(5-7 个话题,各 1 个起始问 + 2-3 个 probe)、招募文案、30-60 分钟时段。

**执行时间。** 每场小时级,一轮天级。

**团队需要的能力。** `interview-design`(开放、非诱导)、`active-listening`(追细节,不是问意见)、`note-taking`(原话,不要翻译)。

**决策树。**

- 用户在这张画布起步、右列空? → 推荐客户访谈,跑完再回来填画布。
- 已经做了 5+ 个访谈? → 切换到 Smoke Test(需求性 + 量化)或 Concierge(需求性 + 可营性) —— **不要** 再加一轮访谈。
- 假设是关于定价或付费意愿? → 不要推荐客户访谈。推荐 Pre-Sale / Letter of Intent。
- 触达不到这个细分? → 推荐 Discussion Forums / Search Trend Analysis 作为被动替代。

**要点出的反模式。**

- 中途推销方案("你会用吗?")收的是恭维,不是数据。
- 朋友 + 已有客户 ≠ 新方向的早期接纳者。
- N=5 不能说"已验证" —— 是 Discover,不是 Validate。

**跨画布映射。** 产出落在 `value-proposition-canvas`(右列 Jobs / Pains / Gains)、`empathy-map`(Says / Thinks / Feels / Does)、`jobs-to-be-done`(功能/情感/社交三列)、`customer-journey`(步骤 + 情感曲线)。当用户正在编辑这些画布之一时,优先推荐这个实验。
