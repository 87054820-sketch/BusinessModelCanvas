# Value Networks and the Impetus to Innovate

> **Core argument:** A value network defines what a firm sees as valuable. The same management system that allocates resources brilliantly for sustaining innovation can systematically reject disruptive opportunities because they look unattractive inside the existing network.

## Why This Chapter Matters

Chapter 1 names the paradox: good management can make great firms fail. This chapter gives the mechanism. Christensen's key idea is the **value network** — the market, customer, cost, and competitive context in which a company decides what matters. Managers do not evaluate technologies in the abstract. They evaluate them through customers, margins, production systems, sales channels, and competitors that already define success. That is why the failure pattern is so hard to correct: the company is not ignoring data; it is obeying the data that its network makes visible.

## Core Concepts

### Value Network

A value network is the context within which a firm identifies and responds to customers' needs, solves problems, procures inputs, reacts to competitors, and strives for profit. It defines the performance attributes customers will pay for, the cost structure that can be supported, and the size of opportunity that looks meaningful. Two firms can look at the same technology and see different truths because they occupy different networks.

### Impetus to Innovate

Inside a value network, there is constant pressure to improve what current customers already value. Sales teams bring customer requests; financial systems compare margins; engineering teams get rewarded for solving high-priority mainstream problems. The impetus to innovate therefore points toward sustaining projects — better capacity, reliability, performance, or cost for the existing customer base.

### Sustaining vs. Disruptive Innovation

Sustaining innovations improve products along dimensions mainstream customers already value. Disruptive innovations initially underperform on those dimensions but offer a different package of attributes — smaller, cheaper, simpler, more convenient, or accessible to nonconsumers. The trap is that the disruptive product looks bad when judged by the old network and valuable when judged by the foothold network.

## Key Arguments

### Incumbents prioritize sustaining innovation structurally

The incumbent's best customers ask for better versions of the current product. Those customers often carry the highest margins and the greatest strategic weight. Resource allocation processes naturally favor projects that satisfy them. This is not managerial stupidity; it is the normal result of a system built to serve a successful value network.

### A technology's value depends on the network judging it

A smaller disk drive, hydraulic excavator, or mini-mill product is not inherently attractive or unattractive. It is attractive to a network that values compactness, simplicity, lower cost, or access, and unattractive to a network that values capacity, heavy-duty power, or premium quality. The same technical object changes meaning when the customer job changes.

## Real-World Cases

### Disk-drive form factors

Smaller drives initially offered lower capacity than mainstream customers needed. A leading drive maker could build them and still decline to commercialize them because its customers wanted capacity, reliability, and cost per megabyte. Entrants, however, sold smaller drives to applications that valued size and power consumption. The lesson is not that incumbents lacked engineering skill; their value network rejected the application.

## Connecting to Practice

When diagnosing a disruptive threat, do not ask only whether the technology is better. Ask: better for whom, on which performance dimension, and inside which cost structure? Map your value network explicitly: best customers, margin requirements, sales incentives, metrics, and performance attributes. Then map the possible foothold network for the new technology. If the new network values what your current one rejects, you are looking at a disruption problem, not a feature-roadmap problem.

## Key Takeaways

1. A value network defines what looks valuable, profitable, and strategically important.
2. Good resource allocation inside one network can be fatal when another network is emerging.
3. Sustaining innovation fits existing customers and metrics; disruptive innovation begins where those metrics undervalue it.
4. Customer listening is powerful for sustaining improvement and dangerous when it is the only lens for disruption.
5. The practical move is to identify the network that values the new attributes now, not to force the technology through the old network.
