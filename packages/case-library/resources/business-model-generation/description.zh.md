# 《Business Model Generation / 商业模式新生代》阅读地图

这本书是 PinGarden 资料库里最基础的一本：它不是只解释一张 `business-model-canvas`，而是说明为什么商业模式应该被当作一个可视化系统来讨论、设计和迭代。

## 这本书解决什么问题

- 团队对“商业模式”的定义不一致：有人讲收入，有人讲产品，有人讲渠道，讨论很容易散。
- BMC 被当作九个孤立格子填写，看不到格子之间的因果关系。
- 案例学习只停留在“这家公司很成功”，没有抽象出可复用的模式。
- 新业务设计缺少从观察、构思、原型、测试到落地的共同语言。

## 目录/阅读地图

1. **Canvas / 画布**：九大模块是全书的入口，重点不是背定义，而是理解客户、价值、交付、收益和成本如何互相支撑。
2. **Patterns / 模式**：把案例抽象成可复用的商业模式结构，包括拆分式业务、长尾、多边平台、免费和开放式商业模式。
3. **Design / 设计**：介绍洞察客户、构思、视觉化、原型、讲故事和情景设计，适合和 PinGarden 的画布编辑工作流结合。
4. **Strategy / 战略**：用商业模式环境、SWOT、蓝海视角和组合视角审视一个模型是否足够稳健。
5. **Process / 流程**：把商业模式创新拆成动员、理解、设计、实施和管理，而不是一次性填完表。

## 和 PinGarden 画布怎么对应

- `business-model-canvas`：核心入口，用来描述一个模式如何创造、传递和获取价值。
- `value-proposition-canvas`：把 BMC 里的客户细分和价值主张展开，避免价值主张写成口号。
- `business-model-environment`：对应书中的战略环境分析，帮助解释为什么某个模式在特定时代成立。
- `portfolio-map`：当公司不只有一个模型时，用来比较现有业务、增长业务和探索业务。

## 建议重点读哪些章节

- 如果你在做**单个公司案例**：先读九大模块，再回到案例的 BMC，把每个便利贴改成更具体的机制描述。
- 如果你在做**模式库**：重点读 Patterns 部分，把书里的模式和 PinGarden 的 `long-tail`、`multi-sided-platforms`、`free`、`open-business-models`、`unbundling-business-models` 对齐。
- 如果你在做**新业务设计**：读 Design 和 Process，把每个想法先变成多个低成本原型，而不是直接写完整方案。
- 如果你在做**战略判断**：读 Strategy，把 BMC 放进外部环境、蓝海空间和组合管理里看。

## 常见误读

- **误读 1：BMC 是商业计划书的简化版。** 实际上它更像团队对商业逻辑的共同假设图，应该频繁修改。
- **误读 2：九个格子填满就完成了。** 更重要的是格子之间是否自洽，例如渠道承诺是否匹配成本结构，关键伙伴是否真的降低风险。
- **误读 3：模式可以直接套用。** 长尾、免费、多边平台等模式只是结构模板，必须重新验证客户、供给、网络效应和变现条件。

## 下一步可以打开

- 想建第一版模型：打开 `business-model-canvas`。
- 想检查客户和价值是否具体：打开 `value-proposition-canvas`。
- 想从案例抽象模式：阅读 `long-tail`、`multi-sided-platforms`、`free`、`open-business-models`。
- 想判断模型受哪些外部力量影响：打开 `business-model-environment`。
