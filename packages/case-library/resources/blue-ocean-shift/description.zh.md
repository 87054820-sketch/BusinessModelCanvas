# 《Blue Ocean Shift / 蓝海战略 2》阅读地图

这本书适合在读完《Blue Ocean Strategy》之后继续使用。前一本更像战略工具箱，这一本更像组织转型手册：如何让一个真实团队从“知道要差异化”走到“愿意并能够做出蓝海转型”。

## 这本书解决什么问题

- 团队画出了新的战略曲线，但组织仍然按照旧指标和旧边界行动。
- 蓝海战略被当成少数高层的创意，而不是一套让更多人参与的流程。
- 非顾客洞察没有进入决策，只是作为调研材料存档。
- 新战略缺少小步验证，导致团队要么过早承诺，要么一直停留在讨论阶段。

## 目录/阅读地图

1. **准备转型**：选择合适的项目范围，组建团队，并建立市场创造的共同语言。
2. **理解当前状态**：用战略布局图看清红海竞争因素，让团队看见“我们为什么被困住”。
3. **想象可到达的未来**：研究非顾客、买方效用和痛点，找到新的需求空间。
4. **找到蓝海选择**：形成多个可选路径，而不是只赌一个创意。
5. **行动与落地**：用公平过程、证据和阶段性承诺降低组织阻力。

## 和 PinGarden 画布怎么对应

- `blue-ocean-strategy-canvas`：记录当前价值曲线和目标价值曲线。
- `customer-journey`：把买方效用、障碍和非顾客流失点放进真实旅程。
- `design-criteria-canvas`：把蓝海选择转化为设计原则和取舍标准。
- `experiment-canvas`：用小实验验证新需求、新触点和新付费逻辑。
- `business-model-canvas`：把蓝海选择转成可运营的商业模式。

## 建议重点读哪些章节

- 如果你在做**从红海到蓝海的转型**：重点读流程章节，把每一步变成团队工作坊。
- 如果你在做**非顾客研究**：关注非顾客分层、买方效用和阻碍购买/使用的因素。
- 如果你在做**内部推动**：读公平过程和组织信心相关内容，避免战略变成“领导拍板”。
- 如果你在做**新业务立项**：把候选蓝海选择接到 `experiment-canvas`，先验证再加大投入。

## 常见误读

- **误读 1：Blue Ocean Shift 只是 Blue Ocean Strategy 的案例版。** 它的重点是转型流程和人的参与。
- **误读 2：只要找到非顾客就有新市场。** 非顾客洞察还需要转成可交付的价值、价格、成本和采用路径。
- **误读 3：组织共识靠宣讲建立。** 书中更强调共同看见事实、共同参与取舍和用证据建立信心。
