# 《The Invincible Company》阅读地图

这本书适合放在 **Portfolio Map / 组合地图** 和 **Innovation Metrics / 创新指标** 之间阅读。它不是单讲某一张画布，而是把商业模式组合、探索/开发、创新指标、创新文化和实验节奏放在同一个管理系统里。

## 这本书解决什么问题

- 公司只优化现有业务，缺少对未来增长选项的系统投资。
- 创新项目很多，但没有统一标准判断风险是否降低、证据是否增强。
- 探索型团队和经营型团队使用同一套 KPI，导致早期项目被过早否定。
- 画布、实验和文化工具互相脱节，无法形成管理闭环。

## 目录/阅读地图

1. **Business Model Portfolio / 商业模式组合**：区分 Exploit 与 Explore，用组合视角看当前业务和未来机会。
2. **Portfolio Map / 组合地图**：把业务放到风险和收益两个维度上，判断应该投入、转向、转入、剥离还是终止。
3. **Explore / 探索**：解释创新旅程、证据积累、项目阶段和常见误区。
4. **Exploit / 开发**：说明成熟业务如何继续改善、保护优势并避免被新模式颠覆。
5. **Innovation Culture / 创新文化**：把组织结构、激励、领导方式和创新实践连接起来。
6. **Innovation Metrics / 创新指标**：用证据、风险降低和组合健康度衡量创新，而不是只看短期收入。

## 和 PinGarden 画布怎么对应

- `portfolio-map`：核心入口，用来比较现有业务、增长业务和探索项目。
- `experiment-canvas`：把 Explore 项目的关键假设转成证据积累路径。
- `innovation-culture-map`：检查组织是否允许探索型项目存活。
- `business-model-canvas`：每个组合条目都应能回到具体商业模式逻辑。

## 建议重点读哪些章节

- 如果你在做**公司级创新组合**：先读 Business Model Portfolio 和 Portfolio Map。
- 如果你在做**创新项目评审**：重点读 Explore 和 Innovation Metrics，把评审标准从“收入多少”改成“风险降低多少”。
- 如果你在做**组织机制设计**：读 Innovation Culture，检查团队是否有时间、预算、授权和保护机制。
- 如果你在做**成熟业务转型**：读 Exploit，理解现有业务如何改善和防御。

## 常见误读

- **误读 1：组合地图只是项目看板。** 它真正的作用是帮助管理层做资源配置和退出决策。
- **误读 2：探索项目也应该马上证明 ROI。** 早期更重要的是证明关键风险正在下降。
- **误读 3：创新文化等于鼓励创意。** 文化要落到治理、激励、指标和组织保护机制上。
