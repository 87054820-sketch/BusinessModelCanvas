# 情景规划的基础

> **核心论点：** 情景规划不是预测，而是一种有纪律的绩效干预：它扩展心智模型、改善不确定性下的决策，并帮助组织在被迫行动之前预演多个可信未来。

## 本章为什么重要

Thomas Chermack 的贡献，是把情景规划从鼓舞人心的未来故事，拉回到基于绩效的组织实践。流行叙述常强调经典案例或富有创造力的未来叙事。本章问一个更难的问题：情景规划到底在个体、团队和系统内部做了什么？答案从认知开始。组织通过决策产生绩效；决策受心智模型塑造；而心智模型面对不确定性时常常过窄。情景规划通过在事件迫使组织行动前拉伸这些模型来改善绩效。

## 核心概念

### 情景规划

情景规划是探索多个可信未来的结构化过程。它不试图预测未来，也不选择最可能的单一预测。相反，它要求管理者检视几个内部一致、战略相关、且差异足以挑战当前假设的未来。产出不只是故事，而是关于可能发生什么以及组织如何回应的更好对话。

### 基于绩效的情景规划

基于绩效的情景规划把流程与组织结果连接起来。它会问：这个情景项目要解决什么绩效问题？需要什么学习？决策应如何改善？结果如何评估？这把情景工作从一次性工作坊变成旨在提升战略敏捷性、决策质量和准备度的干预。

### 心智模型

心智模型是人们关于世界如何运作的内部表征。领导者用心智模型过滤信号、解释变化并选择行动。情景规划通过让假设显性化，并让领导者置身多个可信未来，刻意挑战这些模型。当人们能看见此前忽略的选项、风险和弱信号时，价值就出现了。

## 关键论证

### 情景规划先改变思考，再改善绩效

战略行动源于解释。如果管理者用狭窄模型解释环境，就会错过弱信号或过度押注单一未来。情景通过在行动被迫发生前扩展感知来改善绩效。当这种认知变化改变决策、投资和监控时，它才变成组织价值。

### 不确定性需要多个未来，而不是单一预测

预测常把不确定性压缩成一条预期路径。情景规划保留不确定性，让团队与几个可信未来共同工作。目标不是准确押中某一个未来，而是更少意外、更能适应，并更快识别哪个未来正在出现。

## 案例

### Royal Dutch Shell / Wack 传统

Shell 传统与 Pierre Wack 及后续情景实践者相关，展示了情景如何改变高管感知。它的价值不在于拥有完美预测，而在于管理者曾预演过断裂，因此能更快解释变化。情景规划成为让心智和组织为战略惊讶做好准备的方法。

## 应用启示

每个情景项目都从绩效问题开始。哪个决策、能力或战略不确定性需要更好的思考？然后识别当前塑造组织未来观的假设。第一个交付物不是精美故事，而是心智模型与不确定性的地图。情景随后作为工具，用来挑战这些假设并改善决策。

## 关键要点

1. 情景规划不是预测，而是不确定性下的结构化学习。
2. 最深的产出是改变心智模型，而不是漂亮故事文档。
3. 基于绩效的情景规划把前瞻与可衡量组织价值连接起来。
4. 多个可信未来帮助管理者在现实迫使行动前预演惊讶。
5. 情景项目应从决策或绩效问题开始，而不是为讲故事而讲故事。
