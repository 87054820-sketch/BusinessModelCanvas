# 方法和技术：全面综述

> **核心论点：** 情景规划不是一种方法。不同技术在参与度、严谨性、数据强度、叙事丰富性和决策有用性之间做不同权衡；正确方法取决于组织情境。

## 本章为什么重要

许多团队把情景规划简化为一次二乘二矩阵工作坊。Chermack 展示了这个领域更宽广：有与 Shell 和 GBN 相关的直觉逻辑传统，有趋势冲击和交叉影响等定量方法，有形态分析、场异松弛、法国学派、Futures Group 技术、TAIDA 式流程，以及许多程序性混合方法。本章重要，因为方法选择会塑造组织学到什么。

## 核心概念

### 情景方法分类

情景方法是把不确定性转化为可信未来的有纪律方式。有些方法强调专家判断和叙事；有些强调变量间的定量关系。有些重视广泛参与；有些需要技术专业能力。把它们视为可互换，会削弱实践。

### 直觉逻辑法

直觉逻辑法属于 Shell/Schwartz/GBN 家族。它识别驱动力，按不确定性和影响力排序，选择关键不确定性，构建情景逻辑，并写出帮助管理者讨论启示的叙事。优势是战略对话和假设测试；弱点是高度依赖引导质量和参与者纪律。

### 方法—情境适配

没有一种方法普遍最优。正确方法取决于决策类型、时间跨度、数据可得性、专业能力、时间、文化，以及群体讨论不确定性的准备度。技术上严谨的方法若无法让决策者参与，也会失败；参与式方法若面对需要深度定量建模的决策，也可能失败。

## 关键论证

### 方法选择是战略设计

选择方法不是行政细节。它决定谁参与、什么算证据、不确定性如何被结构化、故事如何生成、决策如何被测试。因此，情景项目应在澄清绩效问题后再选择方法。

### 二乘二矩阵有用但不充分

两个关键不确定性构成的矩阵很有力，因为它创造四个易记未来。但情景规划不能被简化成画象限。团队仍须识别驱动力、判断不确定性和影响力、构建逻辑、写出连贯情节，并测试启示。

## 案例

### Shell / GBN 直觉逻辑法

Shell 传统用叙事情景支持战略对话。团队识别力量、聚类不确定性，并构建挑战高管假设的故事。当目标不是计算预测，而是帮助领导者识别潜在断裂并预演行动时，这种方法最有效。

## 应用启示

选择方法前，先写清决策情境。问题是探索性的还是运营性的？时间跨度短还是长？决策者愿意参与吗？数据丰富还是稀缺？组织需要共享语言、定量敏感性，还是创造性重构？然后选择优势适配任务的方法。不要因为熟悉就默认使用二乘二矩阵。

## 关键要点

1. 情景规划包含多种方法，不是唯一标准工作坊。
2. 直觉逻辑法擅长叙事、参与和战略对话。
3. 定量和形态方法可能适配不同数据或决策情境。
4. 方法选择应跟随决策与绩效问题。
5. 矩阵是工具，不是整个实践。
