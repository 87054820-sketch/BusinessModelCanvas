# 《Value Proposition Design / 价值主张设计》阅读地图

这本书适合在填写 BMC 之后立即阅读，尤其当 `Customer Segments` 和 `Value Proposition` 仍然很空泛时。它把“客户是谁”和“我们给什么价值”拆成可以设计、测试和演进的结构。

## 版本和来源确认

- 早前的本地 `BusinessBooks/Value-proposition-design.pdf` 只有 55 页，仍应按**预览资料**处理。
- 新增的 `BusinessBooks/Value+Proposition+FULL.pdf` 是完整版：PDF 共 323 页，书签目录覆盖前言、Canvas、Design、Test、Evolve、Glossary 和 Index。
- 新增的 EPUB `BusinessBooks/Value+Proposition+Design_+How+to+Create+Products+and+Services+Customers+Want-Wil.epub` 含有干净的章节 HTML 和 Wiley / Strategyzer 版本元数据。
- Amazon/Wiley 的外部书目信息仍作为正式引用：**Value Proposition Design: How to Create Products and Services Customers Want**，Wiley，2014，320 页，ISBN-13 `978-1118968055`。

## 这本书解决什么问题

- 价值主张写成“更快、更便宜、更好体验”，但不知道对应哪个具体客户任务。
- 客户研究只收集意见，没有转化成产品、服务和业务模型假设。
- 团队把客户痛点当成事实，却没有区分观察、解释和待验证假设。
- 实验结果无法回流到价值主张和 BMC。

## 完整版阅读地图

1. **Canvas / 画布**：客户画像由 Jobs、Pains、Gains 组成；价值地图由 Products & Services、Pain Relievers、Gain Creators 组成；Fit 连接客户侧和价值侧。
2. **Design / 设计**：创建多个价值主张原型，使用设计约束，研究客户，比较竞争方案，并把价值主张连接到商业模式。
3. **Test / 测试**：抽取假设，优先验证可能杀死想法的风险，设计实验，记录学习，并避免数据陷阱。
4. **Evolve / 演进**：让组织对齐，持续监测 fit，持续改善，并在客户行为或外部环境变化时主动重塑。

## 完整书案例索引

- `nespresso`：作为“剃刀刀片 / 可重复消耗品”设计约束出现。它说明价值主张往往和收入模式、转换成本、锁定机制不可分割。
- `airbnb`：作为平台设计约束出现。它说明平台每一边都需要独立价值主张，而且只有多边同时成立，平台模型才成立。
- `alibaba-group` / 淘宝：出现在 Evolve 章节。它说明平台价值主张会随着信任、支付、物流、卖家和买家行为变化而演进。
- **Azuri / Eight19**：未来强候选案例。它展示太阳能技术、农村用户痛点、支付方式设计和可行商业模式之间的反复匹配。
- **Owlet**：未来强候选案例。它展示系统化测试、从医院/护士转向焦虑父母的客户转向，以及访谈、登陆页、价格测试如何驱动学习。
- **Hilti、Swatch、Southwest、Apple、Dropbox、Marriott**：可作为设计约束、竞争比较或实验方法示例，但本轮信息不足以进入完整案例库。

## 和 PinGarden 画布怎么对应

- `value-proposition-canvas`：核心画布，用来把客户任务和价值主张逐项对齐。
- `jobs-to-be-done`：把客户画像背后的任务说清楚。
- `empathy-map`：在填写 VPC 前补充更丰富的客户洞察。
- `customer-journey`：把任务、痛点、收益和触点放到时间线上。
- `business-model-canvas`：检查价值主张是否被收入、成本、渠道、伙伴和关键活动支撑。
- `experiment-canvas` 和相关实验方法：把 fit 假设转成可执行验证。

## 建议重点读哪些章节

- 如果你正在**定义客户**：先读 Customer Profile，不要急着写产品功能。
- 如果你正在**做产品方案**：读 Value Map，检查每个功能是否真的缓解痛点或创造收益。
- 如果你正在**连接价值与商业模式**：重点读 Design 2.5 的 Azuri 部分。
- 如果你正在**验证 fit**：重点读 Test 章节和 Owlet 部分。
- 如果你正在**管理演进中的平台**：重点读 Evolve 章节的 Taobao 部分。

## 常见误读

- **误读 1：VPC 是产品功能清单。** 它真正关注的是客户任务与价值创造之间是否匹配。
- **误读 2：客户说想要就是证据。** 访谈和问卷多是弱证据，仍需要行为实验、购买意向或真实使用数据。
- **误读 3：一个产品只有一个客户画像。** 多边平台、企业服务和生态型业务通常需要多张 VPC。
- **误读 4：书中提到品牌就应该建案例。** 只有能充分教学、可画布化、且有足够公开事实支撑的品牌才进入案例库。

## 下一步可以打开

- 想明确客户任务：打开 `jobs-to-be-done`。
- 想整理客户洞察：打开 `empathy-map`。
