# 《Value Proposition Design / 价值主张设计》阅读地图

这本书适合在填写 BMC 之后立即阅读，尤其当 `Customer Segments` 和 `Value Proposition` 仍然很空泛时。它把“客户是谁”和“我们给什么价值”拆成更可验证的结构。

## 版本和来源确认

- 本地 `BusinessBooks/Value-proposition-design.pdf` 只有 55 页，结合 Strategyzer 官方页面的 **Preview & Download PDF** 和 “Download these 100 pages, for free” 说明，应按**预览资料**处理，不应当作完整书。
- Amazon 页面显示完整书名为 **Value Proposition Design: How to Create Products and Services Customers Want**，作者为 Alexander Osterwalder、Yves Pigneur、Gregory/Greg Bernarda、Alan Smith，设计相关署名 Trish Papadakos。
- Amazon 页面显示出版方为 Wiley，出版日期为 2014-10-20，1st edition，英文，**320 页**，ISBN-10 `1118968050`，ISBN-13 `978-1118968055`。
- Wiley/Strategyzer 的公开简介都强调：本书是 `Business Model Generation` 的延伸，使用 `Value Proposition Canvas` 来设计、测试、创建和管理客户真正想要的产品与服务。

## 这本书解决什么问题

- 价值主张写成“更快、更便宜、更好体验”，但不知道对应哪个具体客户任务。
- 客户研究只收集意见，没有转化成产品、服务和业务模型假设。
- 团队把客户痛点当成事实，却没有区分观察、解释和待验证假设。
- 实验结果无法回流到价值主张和 BMC。

## 目录/阅读地图

1. **Canvas / 画布**：客户画像由 Jobs、Pains、Gains 组成；价值地图由 Products & Services、Pain Relievers、Gain Creators 组成。
2. **Design / 设计**：从客户洞察出发，形成多个价值主张原型，而不是一开始就锁定一个方案。
3. **Test / 测试**：把客户画像、价值地图和商业模式假设转化为实验，验证是否真的产生 fit。
4. **Evolve / 演进**：持续根据证据调整客户、价值主张和商业模式，而不是把 VPC 当作一次性输出物。

## 和 PinGarden 画布怎么对应

- `value-proposition-canvas`：核心画布，用来把客户任务和价值主张逐项对齐。
- `empathy-map`：适合在 VPC 之前做定性洞察，帮助理解客户看到、听到、思考和感受到什么。
- `jobs-to-be-done`：把客户任务写得更清楚，避免只用人群标签定义客户。
- `customer-journey`：把任务、痛点和触点放到时间线上，找出价值主张进入客户流程的位置。
- `experiment-canvas`：把 fit 假设转成可执行实验。

## 建议重点读哪些章节

- 如果你正在**定义客户**：先读 Customer Profile，不要急着写产品功能。
- 如果你正在**做产品方案**：读 Value Map，检查每个功能是否真的缓解痛点或创造收益。
- 如果你正在**找产品市场匹配**：读 Fit 和 Test，把“我们相信”改写成假设和证据标准。
- 如果你正在**迭代既有产品**：读 Evolve，检查哪些价值主张已经过时，哪些客户任务被新行为替代。

## 常见误读

- **误读 1：VPC 是产品功能清单。** 它真正关注的是客户任务与价值创造之间是否匹配。
- **误读 2：客户说想要就是证据。** 访谈和问卷多是弱证据，仍需要行为实验、购买意向或真实使用数据。
- **误读 3：一个产品只有一个客户画像。** 多边平台、企业服务和生态型业务通常需要多张 VPC。
