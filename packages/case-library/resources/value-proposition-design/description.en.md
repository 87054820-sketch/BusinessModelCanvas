# Reading map — Value Proposition Design

Read this right after a first BMC draft, especially when `Customer Segments` and `Value Proposition` still sound generic. It turns “who is the customer?” and “what value do we provide?” into a structure that can be tested.

## Edition and source check

- The local `BusinessBooks/Value-proposition-design.pdf` has only 55 pages. Strategyzer's official page is labeled **Preview & Download PDF** and says “Download these 100 pages, for free,” so the local PDF should be treated as preview material, not the full book.
- Amazon lists the full title as **Value Proposition Design: How to Create Products and Services Customers Want**, by Alexander Osterwalder, Yves Pigneur, Gregory/Greg Bernarda, and Alan Smith, with design-related credit for Trish Papadakos.
- Amazon lists Wiley as publisher, publication date October 20, 2014, 1st edition, English, **320 pages**, ISBN-10 `1118968050`, ISBN-13 `978-1118968055`.
- Wiley and Strategyzer summaries both position the book as a sequel/companion to `Business Model Generation`, focused on using `Value Proposition Canvas` to design, test, create, and manage products and services customers want.

## What problem it solves

- Value propositions become generic claims such as faster, cheaper, or better experience.
- Customer research collects opinions but does not become product, service, or business-model hypotheses.
- Teams treat pains as facts without separating observation, interpretation, and assumption.
- Experiment results do not feed back into the VPC and BMC.

## Contents / reading map

1. **Canvas** — Customer Profile: Jobs, Pains, Gains; Value Map: Products & Services, Pain Relievers, Gain Creators.
2. **Design** — create multiple value-proposition prototypes from customer insight instead of locking in one solution too early.
3. **Test** — translate customer, value, and business-model assumptions into experiments.
4. **Evolve** — keep changing the proposition as evidence, customer behavior, and context change.

## How it maps to PinGarden

- `empathy-map` — useful before VPC when the team needs richer qualitative insight.

- `customer-journey` — places jobs, pains, gains, and touchpoints on a timeline.
- `experiment-canvas` — turns fit assumptions into executable tests.

## What to read first

- If you are **defining customers**, start with Customer Profile and avoid jumping to features.
- If you are **designing a product**, read Value Map and check whether each feature relieves a pain or creates a gain.
- If you are **searching for product-market fit**, read Fit and Test; rewrite beliefs as hypotheses with evidence criteria.
- If you are **iterating an existing product**, read Evolve and ask which propositions are now outdated.

## Common misreadings

- **VPC is not a feature list.** It is a fit map between customer work and value creation.
- **Customer statements are not enough evidence.** Interviews and surveys are often weak evidence and need behavioral tests.
- **One product may need many VPCs.** Platforms, enterprise services, and ecosystems usually serve multiple customer profiles.
