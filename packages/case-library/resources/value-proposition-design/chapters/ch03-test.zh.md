# 测试：通过实验降低不确定性

> **核心论点:** 测试不是可选项——它是构建没人想要的东西与构建客户热爱的东西之间的区别。

## 核心概念

### 测试三圈层
内圈（客户任务/痛点/收益）→ 中圈（价值主张）→ 外圈（商业模式）。先测试需求，再测试方案，最后测试规模化。

### 测试卡片
格式：假设（我们认为...），测试（为验证，我们将...），指标（并测量...），判定标准（如果...则正确）。

### 五种数据陷阱
假阳性、假阴性、局部最大值（优化当前框架错过更大机会）、穷尽最大值（市场饱和）、错误数据。

### 实验库
从CT A到预售：号召行动、跟踪广告、唯一链接、数据表MVP、宣传册、故事板、登陆页、产品盒、视频MVP、学习原型、绿野仙踪MVP、管家MVP、众筹、预售。

## 关键要点
1. 先测试需求，再测可行性，最后测商业可行性。
2. 证据强度要与决策风险匹配。
3. 将每次实验视为学习机会——失败假设被证伪也是进展。

## 测试过程的补充说明

测试价值主张的目标不是证明团队正确，而是尽快发现哪些假设错误。每个测试都应连接到一个具体假设：客户是否真的有这个任务，痛点是否足够强烈，收益是否值得追求，痛点缓解项是否可信，客户是否愿意改变行为或付费。若没有明确假设，实验结果再多也很难转化为学习。

证据强度需要分层判断。客户说“我喜欢”是弱证据；客户花时间试用、留下联系方式、推荐给同事、支付订金或改变现有流程，是更强证据。团队应先用低成本实验筛掉明显错误，再逐步提升实验保真度。访谈、调研、纸面原型、可点击原型、烟雾测试和预售不是互相替代，而是按照不确定性递进使用。

学习卡是测试纪律的核心：我们原本相信什么，观察到了什么，由此学到了什么，下一步要做什么。它防止团队只收集数据而不改变决策。若证据反驳假设，正确动作可能是转向、重新设计、缩小范围或停止，而不是继续优化一个客户并不想要的方案。
