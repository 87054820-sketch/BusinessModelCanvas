# NVIDIA CUDA：把 GPU 从图形芯片变成计算平台

## 背景与矛盾

GPU 竞争最初围绕游戏和可视化图形性能。以 CPU 为中心的计算把 GPU 看作专用外设。但许多工作负载需要 CPU 难以高效处理的并行计算。

因此，这个案例不应被理解为普通的产品成功故事，而应被看作一次市场重构。公司并不是在旧规则下做得更好，而是改变了谁是关键买方、哪些因素值得投入、哪些行业惯性可以被移除。新手阅读时，应该先理解旧行业的默认规则，再看这家公司如何换了一条价值曲线。

## 战略动作

NVIDIA 构建 CUDA，让开发者能用 GPU 做通用并行计算，减少对纯游戏图形周期的依赖，提升开发者可及性、库生态和生态锁定。

这个动作同时包含差异化和成本/复杂度突破。公司提高少数对被忽视买方非常重要的因素，同时降低或剔除在位者认为必不可少、但新买方并不真正需要的因素。蓝海战略的重点不是“每一项都更高”，而是曲线形状显著不同。

战略画布要看形状。红海基准通常在传统复杂度、专家门槛或行业惯例上得分更高；NVIDIA CUDA 则在可及性、易用性、个性化价值、成本效率或新需求创造上更高。这个形状就是战略论证。如果公司试图在所有传统因素和所有新因素上同时拉满，往往会重新背上旧行业的成本结构。

## ERRC 逻辑

- **剔除：** GPU 只是图形加速器的边界。
- **减少：** 开发者使用并行硬件的摩擦。
- **提升：** 可编程性、库生态、每瓦性能和生态支持。
- **创造：** 面向 AI 和科学负载的加速计算平台。

## 风险与边界

平台战略需要持续开发者投入，也会面对开放标准、自研芯片和云抽象层竞争。

所以，这个案例不能机械照抄。可迁移的是“匹配”这件事：产品、渠道、运营、收入模型和成本模型都必须服务于同一条价值曲线。

## 可迁移的启发

当硬件被重构为可编程平台时，买方会从设备采购者扩展到开发者和工作负载拥有者，蓝海机会由此出现。
