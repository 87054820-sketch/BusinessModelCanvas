# 战略读法：三层增长与 BCG 增长份额矩阵

## 为什么这个案例需要不止一种组合视角

平安已经是组合管理案例，但业务组合地图主要说明集团必须同时管理 Explore 和 Exploit。战略框架标签真正有用，是因为它们回答不同问题：**三层增长**看当前核心、新增长引擎和未来选项如何随时间迁移；**BCG 增长份额矩阵**看哪些业务提供现金、哪些业务值得投入、哪些业务应收割或终止。

## 三层增长读法

::canvas[three-horizons-map]{}

H1 是金融服务核心：保险、银行、财富管理、风险模型、分销和客户信任。H2 是可能成长为重要增长引擎的平台业务，前提是使用频率和变现证据逐步积累。H3 是围绕数据、AI 服务和新生态循环建立的、有纪律的未来选项。

关键教学点是“迁移”。医疗或车主平台不能因为数字化就自动成功。只有当重复使用、付费需求、供给可行性和核心协同都有证据时，它才从 H3 进入 H2；只有当商业模式和治理方式都可重复时，才可能进一步靠近 H1。

## BCG 增长份额读法

::canvas[bcg-growth-share-matrix]{}

BCG 视角把成熟保险和银行视为现金牛，为被选择的新增长业务提供资金。部分数字生态如果市场增长和相对位置都提升，可能成为明星业务。其他试点仍然只是问题业务：有潜力，但不能自动获得规模化投入。

## 不应过度解读什么

BCG 矩阵不能单独证明某个业务有吸引力。平台生态里的市场份额定义可能模糊，监管也会快速改变经济性。它更适合作为资源分配对话的入口，然后回到组合地图、BMC、实验和证据评分卡判断战略动作是否稳健。
