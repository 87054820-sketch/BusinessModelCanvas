# 如何使用

1. **定义分析单元。** 单一业务、一张 BMC。不是整个集团。行业边界、客户细分、时间窗必须明确写出。
2. **主要活动从左到右填。** 进货物流 → 生产运营 → 出货物流 → 营销与销售 → 售后服务。每个内部写 2-4 个具体子活动（不是「生产运营」——而是「按周排程变动的高混合柔性装配」）。
3. **支持活动从上到下填。** 企业基础设施 → 人力资源管理 → 技术开发 → 采购。这些看起来"无聊"，但通常是最难复制的护城河——它们支撑下方的一切。
4. **给每个活动标颜色。** 黄 = 成本优势活动。蓝 = 差异化优势活动。可以同时成立（少见但真实）。
5. **找出联动。** 用紫色 sticky 标注互相加强的活动（如采购品质 → 生产稳定性）或互相打架的活动（如营销承诺 24 小时服务，但服务只能交付 48 小时）。
6. **标出漏点。** 在成本上涨、差异化失效或能力低于战略所要求底线的活动上贴粉色 sticky。
7. **挑出 2-3 个承重差异化活动。** 能力投入集中在这里。用绿色 sticky 标记。
8. **翻译到 BMC。** 每个承载优势的活动都必须落到 BMC 的关键活动、关键资源或关键伙伴 sticky 上。没落到 BMC 的活动还不是决策。

# 如何解读

- **成本优势占主。** 公司靠规模、标准化或流程纪律赢。通过推进成本前沿（自动化、产能、供应商整合）来守护。注意：长期价格战只是侵蚀利润，不强化护城河。
- **差异化优势占主。** 公司靠品质、服务或品牌赢。守护的方式是让差异化活动持续领先竞争对手，并确保客户依然愿意为这个差异付溢价。
- **混合（多个活动同时双色）。** 可能但少见。Patagonia 是一例——可持续采购既是成本纪律也是品牌差异化。大多数"卡在中间"的尝试都失败；验证联动是否真的复利。
- **大量粉色漏点。** 战略是被描述的，不是被执行的。能力投入是下季度的议程。
- **联动集中在支持活动。** 护城河是组织性的。HRM 和企业基础设施在承重。最难被复制。

# 质量检查表

- 每个活动有 2-4 个具体子活动，不只是一个标签。
- 至少标了 2 条紫色联动——真正的差异化多来自联动。
- 支持活动填写了，没有因为"无聊"被跳过。
- 平台 / 服务业务的进货与出货物流可能几乎为空——这是对的；技术开发和容量采购在主导。
- 每个承重活动（黄或蓝）通过后续 sticky 或备注映射到 BMC 块。
- 利润区写明钱到底从哪来——成本纪律、差异化溢价，还是兼有。

# 常见错误

- 9 个格子画成通用流程图，没有优势上色。
- 单独评估每个活动，跳过联动。
- 因为不如运营有趣就跳过支持活动。
- 服务或平台业务里假装实体进货与出货物流仍是重头戏。
- 分析停在画布上，没翻译成 BMC KA / KR / KP。
- 把"能力"和"当下表现"混为一谈——团队今天擅长生产运营，不代表能扩 10 倍。
