# 证据评分卡

证据评分卡帮助团队判断：已经做过的实验到底证明了什么。它适合放在访谈、落地页、Concierge 测试、试点、预售或销售沟通之后使用，因为这时团队通常已经有很多活动记录，但还没有清楚地回答下一步是加码、转向、继续验证还是终止。

新手先记住一个核心概念：观点还不是证据。证据只有连接了清晰假设、测试设计、可观察行为和决策，才真正有用。客户说“我喜欢”通常弱于客户改变工作流、投入时间、提供数据、签署意向、预付费用或真实使用。

## 何时使用

当项目已经跑过一轮或多轮实验，下一步要回答“现在该做什么”时使用它。它特别适合探索团队、创业团队、内部创新委员会和产品小组，用来比较弱信号，而不是把每次测试都包装成成功案例。

## 何时不适合

如果团队还说不清正在验证哪条关键假设，不要先用这张画布。商业模式还没成形时，先用 BMC 或 VPC；还没有设计任何实验时，先用 Experiment Canvas；如果问题是多个项目之间如何分配资源，应该先为每个项目填一张评分卡，再进入 Portfolio Map。

## 读前概念

- **关键假设：** 如果为假，就会让这个想法失败的核心判断。
- **证据强度：** 测试离真实行为、真实承诺或真实经济性有多近。
- **学习速度：** 团队多久获得一次能改变决策的学习结果。
- **风险下降：** 需求性、可行性、可营性或适应性风险是否降低。
- **转向或继续：** 改方向、继续、补证据、加码、转入或终止的决策。
