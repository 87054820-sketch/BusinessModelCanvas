# Portfolio Decision

This module turns evidence into resource allocation. The decision may be persevere, pivot, test more, transfer learning, fund, or retire. The point is to avoid innovation theatre: a project should not keep consuming time because activity feels positive.

## Good Answer vs Weak Answer

A good answer states the decision, the evidence behind it, and the condition for revisiting it. A weak answer says "continue learning" with no budget, owner, or decision trigger.

## Common Pitfalls

- Funding a project because the story is exciting rather than because risk decreased.
- Killing a project because one weak test failed.
- Ignoring that learning may be transferable even when the specific idea should stop.

## Sticky Examples

- "Persevere: fund two-week concierge test; retire if fewer than 2 paid commitments."
- "Pivot: keep workflow insight, switch from SMB to mid-market buyer."
- "Transfer: pricing evidence useful for adjacent reporting project."

## Transfer Questions

- What resource changes now: budget, people, timebox, or leadership attention?
- Would another project in the portfolio benefit more from this learning?
