# 怎么读这张画布

VPC 有两个互相对望的部分:

- **客户画像** (右侧,圆形) —— 不论你卖什么,客户那一侧固有的事实。
- **价值地图** (左侧,方形) —— 不论卖给谁,你的产品/服务这一侧的事实。

两侧"对得上",就是 fit。

## 客户画像

- **客户任务** — 客户在工作或生活中想完成的事。功能性、社交性、情感性
  任务都算。
- **痛点** — 客户希望规避的糟糕结果、障碍与风险。
- **收益** — 客户希望获得的结果与好处:必须有的、预期会有的、希望有的、
  意外惊喜的。

好的画像是具体的。"节省时间"只是个占位词;"30 分钟以内完成本周预测、
中途不让表格崩溃"才是一个任务。

## 价值地图

- **产品与服务** — 你真正摆在客户面前的东西。
- **痛点缓解** — 具体来说,你的提供物如何消除客户某一个具体痛点。
- **收益创造** — 具体来说,你的提供物如何带来客户想要的某一项具体收益。

痛点缓解 / 收益创造 都必须明确指向画像里**已经写出来的某一项**。否则,
那只是"找问题的功能"。

## 怎么达到 fit

1. **画出一个客户细分的画像**,不要混入其他细分。
2. **画出你正在考虑的价值地图**。
3. **逐项对照**。每一项缓解措施要解决画像里写过的某个痛点;每一项收益
   创造要带来画像里写过的某项收益。
4. **标出对不上的项**——画像那侧没被对上的,是尚未满足的机会;价值地图
   那侧没被对上的,是没有立得住理由的功能。
5. **拿真实客户验证**,再决定是否加大投入。在拿到证据之前,fit 始终
   只是假设。

## 常见误区

- 为了把已经做出来的功能"圆回来",反向编造痛点和收益。
- 把好几个客户细分塞进同一张画像里,导致痛点和收益列得彼此矛盾。
- 把"产品与服务"那一栏当作最重要的——其实它信息量最少,真正承载论证
  的是缓解措施和收益创造。

## 延伸阅读

- 《价值主张设计》——Osterwalder, Pigneur, Bernarda, Smith。
- 配套使用 **商业模式画布**:从单个细分的 fit,放大到整体业务。
