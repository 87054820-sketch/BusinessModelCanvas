# 怎么读这张画布

VPC 有两个互相对望的部分：

- **客户画像**（右侧，圆形）—— 客户那一侧的事实：他们想完成什么、哪里痛、想得到什么。
- **价值地图**（左侧，方形）—— 你的产品/服务这一侧的事实：你提供什么、缓解什么痛点、创造什么收益。

两侧对得上，就是 fit。强价值主张不是「功能很多」，而是「少数关键功能精准命中客户最在意的任务、痛点和收益」。

## 客户画像

先从客户开始，不要从产品开始。

- **客户任务** — 客户在工作或生活中想完成的事。功能性、社交性、情感性任务，以及基本需求都算。
- **痛点** — 客户在完成任务前、中、后希望避免的糟糕结果、障碍、成本、风险和负面情绪。
- **收益** — 客户期待、渴望，或会感到惊喜的结果，包括功能收益、社会收益、积极情绪、成本或时间节省。

好的画像是具体的。「节省时间」只是占位词；「30 分钟以内完成本周预测，中途不让表格崩溃」才是一个可讨论、可验证的任务。写完后要排序：哪些任务最重要？哪些痛点最强、最高频？哪些收益真正影响客户选择？

## 价值地图

价值地图不是产品清单，而是对客户画像的回应机制。

- **产品与服务** — 你真正摆在客户面前的东西，可以是有形产品、数字产品、服务、权利、金融产品等。
- **痛点缓解方案** — 你的提供物如何减少、消除或规避客户侧某个具体痛点。
- **收益创造方案** — 你的提供物如何带来客户期待、渴望或惊喜的某项具体收益。

痛点缓解 / 收益创造必须明确指向画像里已经写出来的某一项。否则，那只是「找问题的功能」。

## 三层 fit

1. **Problem-Solution Fit**：你是否真的回应了客户的关键任务、痛点和收益？
2. **Product-Market Fit**：真实市场中的客户是否需要、使用并认可你的价值主张？
3. **Business Model Fit**：这个价值主张是否能嵌入一个可盈利、可扩展、可持续的商业模式？

VPC 主要帮助你寻找前两层 fit，但不要忘记最后还要回到商业模式画布，验证整体业务是否成立。

## 怎么使用和测试

1. **只选一个客户细分**，不要把多个群体混在同一张画像里。
2. **先画客户画像**，再画价值地图。不要为了已有功能反向编造痛点。
3. **逐项对照**：每一项缓解方案要对应一个具体痛点；每一项收益创造要对应一个具体收益。
4. **标出未匹配项**：客户侧未被回应的是机会；价值地图侧未被匹配的是缺少理由的功能。
5. **拿证据验证**：用访谈、关键词广告、着陆页、功能优先级测试或购买意愿测试验证假设。

可以按三个层次测试：先测试客户画像是否真实存在（Circle），再测试价值地图是否打动客户（Square），最后测试客户是否愿意付费或投入转换成本（Rectangle）。

## 常见误区

- 把好几个客户细分塞进同一张画像，导致任务、痛点和收益互相冲突。
- 只关注功能性任务，忽略社会性和情感性任务。
- 为了把已经做出来的功能「圆回来」，反向编造痛点和收益。
- 试图解决所有痛点和收益，而不是聚焦最重要、最能差异化的少数点。
- 把「产品与服务」当成最重要的区块——真正承载论证的是痛点缓解和收益创造。

## 延伸阅读

- 《价值主张设计》——Osterwalder, Pigneur, Bernarda, Smith。
- 配套使用 **Ad-Lib 价值主张速写**：把展开后的画布收敛成一句可比较、可测试的价值主张。
- 配套使用 **商业模式画布**：从单个客户细分的 fit，放大到整体业务模型。
