# 使用步骤

1. **定义组合单元。** 先决定每张便签代表业务单元、产品线、平台、区域还是创新项目，不能混用层级。
2. **先放 H1。** 列出今天创造收入、利润、数据、渠道或能力的核心业务。
3. **再放 H2。** 加入已经有证据、但尚未成为核心业务的新增长引擎。
4. **最后放 H3。** 加入仍以假设为主、但可能打开未来空间的选项。
5. **写清迁移条件。** 对每个 H3 或 H2 项目，说明需要什么证据、能力、资金或治理变化，才能进入下一层级。
6. **记录风险和证据。** 把未解决的假设放进“证据与风险”，不要把不确定性藏在层级列里。
7. **转成组合动作。** 决定投入、防守、扩张、转入、转向、暂停或终止。

# 如何解读

健康的三层增长地图既有足够 H1 为公司供血，也有足够 H2 形成可见的下一代增长，还保留足够 H3 维持未来选项。关键不只是项目在哪一列，而是它是否有可信的迁移路径。

需要看风险/收益位置时，用 `portfolio-map`。需要看时间节奏、投入节奏和组织交接时，用三层增长地图。

# 质量检查

- 每个项目处在同一分析层级。
- H1 是真实业务，不是模糊能力。
- H2 有证据，而不只是高层热情。
- H3 有学习目标，而不是科技口号。
- 每个迁移动作都有负责人、触发条件和下次复盘时间。
- H2/H3 假设能连接到实验画布或证据评分卡。

# 常见误用

- 把 H1、H2、H3 当作固定年份。它们是成熟度层级，不是日历年份。
- 用流行技术填满 H3，却没有客户或商业模式假设。
- 因为 H1 指标压倒一切，导致 H2 长期缺资金。
- 把没有可复用证据的项目称为 H2。
- 忘记探索团队到运营团队的组织交接。
- 只画一次地图，从不跟踪项目移动。
