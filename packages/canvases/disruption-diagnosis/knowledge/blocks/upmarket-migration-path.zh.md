# 向上迁移路径 (Upmarket migration path)

颠覆故事如果没有轨迹就是不完整的。立足点本身只是一个利基；让它成为颠覆的是进入者**随时间向高端迁移**——沿主流维度改进，同时保留立足点侧优势。等在位者注意到时，颠覆者已穿过主流"够用线"并握住客户关系。

没有可信的向上迁移路径，你拿到的是利基业务，不是颠覆。

## 核心问题

- 立足点之后的下一步 2-3 个客户群是谁？写具体。
- 每个下一步客户群所需的产品能力是什么？（具体功能、性能水平、认证。）
- 沿主流维度改进的速度有多快？引用一个性能轨迹速率。
- 你的性能轨迹什么时候穿过每个下一步客户群的"够用线"？
- 每一次下一步迁移是保留还是侵蚀了你的立足点优势（价格、简单、可获得）？

## 性能轨迹穿越

Christensen 第 9 章框架：客户对每个维度的性能都有一条"够用线"。在位者的性能轨迹通常**超出**主流的够用线（过度满足）。颠覆者的轨迹起步在够用线之下但向上爬。穿过之后，颠覆者在主流市场具有竞争力——在位者的过度满足成为负担。

## 例子

- **Nucor 钢铁 (1970-1995)**："钢筋 → 棒材 → 结构钢 → 板材。25 年，每约 5 年一次钢种迁移。到 1995 年迷你厂在高端板材上与 Bethlehem 的核心业务正面竞争。"
- **Transsion 非洲 (2010-2025)**："功能机 → 入门 Android → 中端 Android → 新兴主流的智能功能混合机。现以同样剧本扩展到拉美 + 南亚。"
- **Carvana (2013-2025)**："2017 年前以旧换新 + 入门二手车 → 认证二手 → 准新车 → 准车队级车辆。每一步都保留贩卖机 + 7 天退车。"
- **本田 Supercub 上移 (1960-1985)**："50cc → 90cc → 250cc → 500cc → CB750。25 年触及 Harley 的主流。等 Harley 注意到时，Honda 已拥有客户。"

## 它如何流向画布其它块

向上迁移路径成为 `three-horizons-map` 的覆盖层（H1 = 立足点、H2 = 下一群、H3 = 主流挑战）和 `portfolio-map` 的 pin 迁移轨迹。如果这条路径缺失，候选应被重分类为新市场进入或利基，而非颠覆。
