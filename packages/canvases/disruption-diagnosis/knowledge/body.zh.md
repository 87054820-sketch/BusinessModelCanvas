# 如何使用

1. **写出候选假设。** "我们相信 [产品 / 服务] 从 [低端 / 新市场] 立足点颠覆 [在位者 / 行业]。" 把立足点客户是谁说具体。
2. **填立足点客户。** 是被过度满足的主流（低端颠覆）还是不消费者（新市场颠覆）？写细分 + 地理 + 场合。如果你说不出真实可识别的立足点，这就不是颠覆。
3. **填更差的维度。** 写出**具体**的主流维度，颠覆者在该维度上起步更差。量化差距。如果颠覆者在主流维度上比在位者更好，那是维持性创新——停止。
4. **填可接受的维度。** 立足点客户换来了什么，使他们愿意接受上面那个"更差"？价格、简单、可获得、便利、场景匹配？他们的交换意愿就是立足点的承重假设。
5. **填在位者理性忽视理由。** 在位者**正确地**告诉自己不出手的原因？典型：利润薄、市场小、自家高利润产品被蚕食的风险。如果在位者既能又愿意出手，你在竞争，不在颠覆。
6. **填向上迁移路径。** 具体的下一步客户群和产品里程碑。不是"以后扩展"——写出客户群 B、客户群 C，以及每一步所需的产品能力。
7. **填最致命假设。** 从以下选：立足点意愿、改进轨迹、在位者继续忽视的时长、组织隔离能力。每个都必须可被证伪，带指标。
8. **应用严格判定备注（橙色）。** 标注 Christensen 三段标准是否全部成立。任意一条不成立，诚实改标签：维持性 / 新市场进入 / 直接竞争。都是有效动作——只是不是颠覆。

# 如何解读结果

- **三段严格判定全部通过。** 你有了一个真正的颠覆候选。打开 `portfolio-map` 把押注放在 Explore 深处；打开 `three-horizons-map` 定位为 H3；把单元从母公司成熟业务指标里组织上隔离开来。
- **立足点通过，劣势不成立（你已经更好）。** 维持性创新。用 BMC + VPC 打磨价值主张；通过优于在位者的产品正面竞争。
- **立足点通过，向上迁移不清晰。** 新市场进入而非颠覆（暂时不是）。留在立足点深耕，推迟"向上迁移"叙事直到客户证据支持。
- **没有立足点（你在抢在位者当下的主流）。** 直接竞争。用五力和价值链找优势；不要假装颠覆剧本适用。
- **最致命假设是"在位者不会出手"。** 这个假设最脆弱。规划好他们出手那天的应对，包括 Christensen 第 5-6 章的组织隔离杠杆。

# 质量检查表

- 每个区域有明确具体（细分、维度、数字、截止日），而非抽象。
- 严格判定备注已填——橙色 sticky 对 Christensen 每条标准给出评分。
- "更差的维度"诚实。写不出真实的劣势，就没有颠覆。
- 向上迁移至少写了 2 个具体的下一步客户群。
- 最致命假设可被证伪，不是愿望。
- 候选已显式与蓝海 / 维持性 / 新市场进入对比过——选了颠覆是有理由的。

# 常见错误

- 把每个新东西都叫"颠覆性"。Uber 案例是教科书级误用；Christensen 本人论证它在出租车里是维持性创新。
- 把新技术与颠覆性创新混为一谈。新技术用在主流客户的现有维度上仍是维持性，无论多新。
- 跳过立足点判定——没有过度满足或不消费的客户基础，你拿到的就是直接竞争。
- 把向上迁移写成一厢情愿，没有用客户群证据支撑。
- 把颠覆性押注塞在母公司成熟业务利润指标下。Christensen 的组织处方是隔离。
- 忘了在位者也可能通过收购回应。严格判定仍适用，但退出路径不同。
