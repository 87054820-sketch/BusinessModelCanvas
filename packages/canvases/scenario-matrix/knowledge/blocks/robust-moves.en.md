# Robust Moves / Early Signals

Robust moves are actions that make sense across several futures. They are different from optimal moves for one favored forecast. Early signals are observable changes that tell the team which scenario may be forming, so the matrix can be updated instead of forgotten.

## Good Answer vs Weak Answer

A good answer separates robust moves, contingent bets, and signals. A weak answer lists generic actions like "monitor market" or "stay agile."

## Common Pitfalls

- Choosing only no-regret moves and avoiding real strategic bets.
- Monitoring signals that no one owns.
- Treating scenarios as a workshop deliverable rather than a decision routine.

## Sticky Examples

- "Robust: keep data export architecture modular across all futures."
- "Contingent: partner with licensed incumbent if restrictive regulation signal appears."
- "Signal: number of enterprise procurement freezes; platform API policy changes; regulator sandbox approvals."

## Transfer Questions

- Which move is useful in at least three scenarios?
- Who owns each signal, and what decision changes when it appears?
