# Critical Uncertainty A

This module names the first axis of the matrix. It should be high-impact, unresolved, and independent enough to create different strategic choices. Good scenario work starts by separating predetermined trends from true uncertainty.

## Good Answer vs Weak Answer

A good answer names two credible end states and explains why the uncertainty matters. A weak answer is a generic concern such as "market changes" or one half of a trend that is already moving in a clear direction.

## Common Pitfalls

- Choosing an uncertainty the team can control directly.
- Choosing a low-impact variable because it is easy to discuss.
- Writing "good demand vs bad demand" instead of a real external uncertainty.

## Sticky Examples

- "Regulation opens data access ↔ regulation restricts data sharing."
- "Enterprise AI budgets expand ↔ budgets freeze under security review."

## Transfer Questions

- If this axis resolves one way, which BMC block changes first?
- What early signal would tell you which end state is forming?
