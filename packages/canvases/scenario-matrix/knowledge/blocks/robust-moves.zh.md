# 稳健动作 / 早期信号

稳健动作是在多个未来中都值得做的行动。它不同于只服务于某个最偏好预测的最优动作。早期信号是可观察变化，用来提示哪个情景可能正在形成，让矩阵可以被更新，而不是被遗忘。

## 好答案 vs 弱答案

好答案会区分稳健动作、条件性押注和信号。弱答案只写“持续观察市场”或“保持敏捷”这类泛泛动作。

## 常见误区

- 只选无悔动作，回避真正的战略押注。
- 监测信号没有负责人。
- 把情景当成工作坊交付物，而不是决策机制。

## 便签示例

- “稳健：在所有未来中保持数据导出架构模块化。”
- “条件押注：若监管收紧信号出现，就与持牌巨头合作。”
- “信号：企业采购冻结数量、平台 API 政策变化、监管沙盒批准数量。”

## 迁移问题

- 哪个动作至少在三个情景中都有价值？
- 每个信号由谁负责？它出现时会改变哪个决策？
