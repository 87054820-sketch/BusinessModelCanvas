# 平台手册

## 输入

带着平台所有者、需求侧参与方、供给侧或互补方、价值单元、匹配或发现流程、信任机制、收费模型，以及目前关于“哪一边最难启动”的证据来填。

## 输出

输出是一张解释生态系统如何创造、传递和获取价值的地图。好的地图会说明核心交互是什么、网络效应从哪里来、哪一边需要补贴或种子供给、什么治理机制保证质量，以及变现如何避免伤害参与。

## 填写顺序

1. **先写参与方。** 先命名需求侧和供给侧，再写功能。
2. **定义核心交互。** 谁创造价值单元，谁消费它，平台用什么过滤或匹配机制连接双方？
3. **映射网络效应。** 区分同边效应、跨边效应、负向效应和多归属。
4. **补治理与信任。** 规则、评价、身份、质量控制、争议处理和激励都是模型的一部分，不是运营细节。
5. **测试变现张力。** 向错误的一边、错误的时点或错误的环节收费，都可能降低流动性。
6. **写冷启动风险。** 判断必须先种哪一边，以及用什么不可规模化动作创造第一批有价值交互。

## 推荐案例

Uber 和 Airbnb 适合理解冷启动、信任和多边流动性。Visa 可以帮助理解支付网络效应和商户/消费者采用。NVIDIA CUDA 则展示开发者生态中工具、硬件和互补软件如何相互强化。

## 下一张画布

回到 Business Model Canvas 检查收入、成本、伙伴和资源。关于网络效应、供给获取或付费意愿的判断需要证据时，进入 Evidence Scorecard。若某一边参与方动机不清楚，为那一边单独做 Value Proposition Canvas。

## 常见误读

- “我们有两类用户”不等于平台；必须有能创造价值的交互。
- 网络效应可能是负的：更多供给可能拉低质量，更多需求可能造成拥堵。
- 变现不只是收入设计，它会改变激励，甚至伤害核心交互。
