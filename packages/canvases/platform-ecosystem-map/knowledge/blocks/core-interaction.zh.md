# 核心交互

核心交互是让平台产生价值的最小、可重复交换。平台架构通常包含参与方、价值单元，以及过滤或匹配机制。如果你不能用一句话描述核心交互，这个平台很可能还只是功能堆叠。

## 好答案 vs 弱答案

好答案会说明谁生产价值单元、谁消费它，以及平台如何帮助匹配、过滤或交易。弱答案只描述功能清单或品牌承诺。

## 常见误区

- 把注册 onboarding 当成核心交互。
- 忘记价值单元，例如一次行程、一条房源、一次 API 调用、一个应用、一条评价或一次支付授权。
- 为少见的高价值交互设计，而忽略日常可重复交互。

## 便签示例

- “房东发布房源；旅客搜索、预订、入住并评价。”
- “司机接受行程请求；乘客完成后支付并评分。”
- “开发者发布兼容库；CUDA 用户在 GPU 工作流中采用。”

## 迁移问题

- 第一笔成功交互必须发生哪些事情？
- 在添加新功能前，核心交互的哪一段最应该被简化？
