# 网络效应

网络效应解释参与者增加如何改变价值。同边效应发生在同一边，例如更多开发者吸引更多开发者；跨边效应发生在不同边之间，例如更多司机缩短乘客等待时间。负向效应也重要：拥堵、垃圾信息、低质量供给或供应者之间竞争过度，都可能降低价值。

## 好答案 vs 弱答案

好答案会写清方向、机制和指标。弱答案只写“有网络效应”，好像规模本身就足够。

## 常见误区

- 把病毒传播误认为网络效应。
- 在参与者可以同时使用多个平台时，忽略多归属。
- 认为每一个新增参与者都有价值，即使质量正在下降。

## 便签示例

- “跨边：更多司机 -> 等待时间更短 -> 更多乘客。”
- “同边：更多开发者 -> 更多可复用库 -> 开发者采用更容易。”
- “负向：低质量房源增加 -> 信任下降 -> 预订减少。”

## 迁移问题

- 什么指标能证明这个循环正在变强？
- 增长在哪里可能造成拥堵或质量问题？
