# Network Effects

Network effects explain how participation changes value. Same-side effects happen within one side, such as more developers attracting more developers. Cross-side effects happen across sides, such as more drivers reducing wait time for riders. Negative effects also matter: congestion, spam, low-quality supply, or competition among suppliers can reduce value.

## Good Answer vs Weak Answer

A good answer names direction, mechanism, and metric. A weak answer simply writes "network effects" as if scale itself were enough.

## Common Pitfalls

- Confusing virality with network effects.
- Ignoring multi-homing when participants can use several platforms.
- Treating every additional participant as valuable, even when quality falls.

## Sticky Examples

- "Cross-side: more drivers -> shorter wait time -> more riders."
- "Same-side: more developers -> more reusable libraries -> easier developer adoption."
- "Negative: more low-quality listings -> lower trust -> fewer bookings."

## Transfer Questions

- What metric proves the loop is getting stronger?
- Where could growth create congestion or quality problems?
