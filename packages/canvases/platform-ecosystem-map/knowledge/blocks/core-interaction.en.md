# Core Interaction

The core interaction is the smallest repeatable exchange that makes the platform valuable. Platform architecture usually contains participants, a value unit, and a filter or matching mechanism. If you cannot describe the core interaction in one sentence, the platform is probably still a pile of features.

## Good Answer vs Weak Answer

A good answer states who produces the value unit, who consumes it, and how the platform helps match, filter, or transact. A weak answer describes a feature list or brand promise.

## Common Pitfalls

- Confusing onboarding with the core interaction.
- Forgetting the value unit, such as a ride, listing, API call, app, review, or payment authorization.
- Designing for a rare high-value interaction while ignoring the repeatable everyday one.

## Sticky Examples

- "Hosts publish listings; travelers search, book, stay, and review."
- "Drivers accept ride requests; riders pay and rate after completion."
- "Developers publish compatible libraries; CUDA users adopt them inside GPU workflows."

## Transfer Questions

- What must happen for the first successful interaction to occur?
- Which part of the interaction should be made easier before adding new features?
