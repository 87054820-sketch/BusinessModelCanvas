# 冷启动 / 风险

冷启动是鸡生蛋问题：需求等供给，供给等需求。平台启动通常需要种子供给、补贴、借用既有网络、聚焦微市场、引入标杆参与者，或暂时用人工方式完成交互。

## 好答案 vs 弱答案

好答案会写清最难激活的一边、第一块市场边界、不可规模化的种子动作，以及可能击穿信任的风险。弱答案只写“增长投放”，却没有解决流动性。

## 常见误区

- 一上来全国铺开，而不是先赢下一个高密度微市场。
- 两边都补贴，却没有形成有机价值的路径。
- 等规模起来后才处理欺诈、监管、安全或舆论反噬。

## 便签示例

- “先做机场到市中心通勤；在高峰时段种子司机供给。”
- “付费拉需求前，先在一个社区招募 50 个高质量房东。”
- “CUDA：先种开发者工具和文档，再期待第三方生态深度。”
