# 情景矩阵

## 何时使用

当用户已有外部信号，并需要把不确定性转成多个可信未来来压力测试战略时使用。

## 填写顺序

1. 定义焦点战略议题。
2. 从 `business-model-environment` 带入信号。
3. 选择两个相互独立的关键不确定性。
4. 写出每个不确定性的两个端点。
5. 写出四个具有不同战略逻辑的情景。
6. 在每个情景中测试 BMC、组合押注和设计准则。
7. 记录稳健动作和早期信号。

## 红旗信号

- 写成好/中/坏版本，而不是不同未来。
- 没有 BMC 或组合含义。
- 没有早期信号。
- 后续监测没有负责人。

## Blocks

`zoneId` 是 API 的稳定标识符,`pingarden canvas write` payload 必须用它,不要翻译。

### `uncertainty-a` — 关键不确定性 A

**Prompt** — 选择一个会显著改变战略选择的不确定性，并写出两端状态。

**Example** — 监管开放 ↔ 监管收紧

**Quality bar** — 这个模块写第一条矩阵轴。 它必须影响大、尚未确定，并且足够独立，能创造不同战略选择。 好的情景工作首先要把预定趋势和真正的不确定性分开。 好答案会写出两个可信端点，并解释为什么这个不确定性重要。

### `uncertainty-b` — 关键不确定性 B

**Prompt** — 选择第二个独立的不确定性，避免和 A 描述同一件事。

**Example** — 技术成熟快 ↔ 技术成熟慢

**Quality bar** — 第二条轴应该和第一条轴相互独立。 如果两条轴描述的是同一股力量，四个未来会坍缩成重复故事。 独立性让矩阵有学习价值：团队能比较不同组合，而不是简单排列乐观程度。 好答案会和不确定性 A 形成有意义的交叉。

### `scenario-1` — 情景 1

**Prompt** — 在 A 的一端和 B 的一端同时成立时，市场会怎样？BMC 哪些模块承压？

**Example** — 写出情景名称、关键信号、机会、威胁和商业模式影响

**Quality bar** — 每个情景都应该是一个可信世界，而不是一个标签。 写一段短故事，说明在两条轴的这个组合下，市场结构、客户行为、竞争者动作、渠道变化、伙伴约束、收入逻辑和成本压力会怎样。 好答案至少会改变一个战略选择。

### `scenario-2` — 情景 2

**Prompt** — 写出第二个组合情景，重点看客户、渠道、伙伴或收入逻辑如何变化。

**Example** — 哪些价值主张更强？哪些成本结构会失控？

**Quality bar** — 情景 2 不应该只是情景 1 的镜像。 用它探索另一套战略逻辑：谁获得权力、谁失去议价能力、什么变得稀缺、哪些客户任务变得紧急。 好答案会把外部条件和商业模式影响连接起来。 弱答案在每个象限都给出同一个建议。

### `scenario-3` — 情景 3

**Prompt** — 写出第三个组合情景，不要把它简单写成悲观版本。

**Example** — 这个未来中谁受益？谁受损？我们有哪些期权？

**Quality bar** — 情景 3 常常帮助团队发现隐藏假设。 不要把它简单写成“坏”。 要问：当两条轴这样组合时，会出现哪些新赢家、替代方案、约束或非顾客？ 好答案会暴露一条不同的价值路径或失败路径。 弱答案只写收入下降、成本上升或客户延迟。

### `scenario-4` — 情景 4

**Prompt** — 写出第四个组合情景，并找出它和其他情景最不同的战略逻辑。

**Example** — 哪些假设需要最早验证？哪些信号值得监测？

**Quality bar** — 情景 4 完成整个矩阵，并迫使团队比较四个未来。 问题不是它是否最可能，而是它是否足够可信，能暴露盲点或创造战略选项。 好答案会产生和另外三个情景不同的影响。 弱答案只是换了形容词，却保持同一套市场逻辑。

### `robust-moves` — 稳健动作 / 早期信号

**Prompt** — 哪些动作在多个情景下都值得做？哪些外部信号会提示情景正在成形？

**Example** — 保留轻资产渠道期权

**Quality bar** — 稳健动作是在多个未来中都值得做的行动。 它不同于只服务于某个最偏好预测的最优动作。 早期信号是可观察变化，用来提示哪个情景可能正在形成，让矩阵可以被更新，而不是被遗忘。 好答案会区分稳健动作、条件性押注和信号。

## Colour legend

- `0` — **信号**: 外部趋势、弱信号或触发事件。
- `1` — **情景影响**: 这个未来会如何改变商业模式。
- `2` — **稳健动作**: 在多个情景下都值得做的动作。
