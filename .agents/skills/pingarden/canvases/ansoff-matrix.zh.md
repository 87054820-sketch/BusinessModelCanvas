# 安索夫矩阵

## 一句话总结

单业务的方向选择画布。给每个增长周期选一个象限——4 象限按执行风险排序（渗透 < 市场开发 ≈ 产品开发 < 多元化）。

## 填写顺序

1. 先标出当前位置（现有产品 × 现有市场）——锚定分析。
2. 头脑风暴 2–4 个**候选增长动作**，每个放进一个象限。
3. 对每个候选写明：维持不变的部分（蓝）、必须改的部分（绿）、最致命假设（粉）。
4. **增长理由**最后填——对比候选，每个周期承诺一个象限。

## 跨块不变量

- 每张画布一个业务。多业务组合先做 BCG，再对每个业务用 Ansoff。
- 不要在同一时期把两个象限叠加在同一业务上，除非显式有容量（紫色 sticky）。
- 多元化必须有清晰的邻近性故事——新旧之间共享什么？没共享就砍掉。
- 市场开发（同一产品、新买家）和产品开发（新产品、同一买家）看起来像但最致命假设完全不同——不要合并。

## 反模式

- 把小功能增加叫「多元化」——大多数其实是产品开发。
- 把 Ansoff 当一次性象限选择，而不是周期性决策（Spotify、Amazon 这样的案例 16-25 年里走过 4 个象限）。
- 因为其它象限「无聊」就选多元化，却没说清共享了什么。
- 跳过「BMC 哪些块必须改」的映射——增长没配套资源。
- 把 Ansoff 与 BCG 混用：Ansoff 是向前的，BCG 是快照；Ansoff 是单业务，BCG 是组合。

## 语气

具体的名字——产品和市场，不是品类。「为现有客户推出新产品线」不够；写明「在现有女性客户造型订阅渠道里推出男装产品线，目标 2026 客户群」。数字和具名群组。

## Blocks

`zoneId` 是 API 的稳定标识符,`pingarden canvas write` payload 必须用它,不要翻译。

### `market-development` — 市场开发

**Prompt** — 现有产品 × 新市场。把同样的产品带到新地域、新人群或新使用场合。风险：中等——产品已被验证，客户还没有。

**Example** — 地理扩张进入新国家

**Quality bar** — 现有产品 × 新市场：把同一产品带到新地理、新细分或新使用场合。 中等风险——产品已被验证，客户没有。 最致命假设是新市场是否真的需要这个产品（按你目前提供的形式）。

### `diversification` — 多元化

**Prompt** — 新产品 × 新市场。两个轴都动。风险：最高——通常只在核心成熟且有真实邻近性，或客户 / 产品至少共享一层底盘时才成立。

**Example** — 亚马逊推出 AWS（新产品 + 新买家）

**Quality bar** — 新产品 × 新市场：两个轴都动。 风险最高的象限。 Ansoff (1957) 明确指出多元化"要求同时偏离当下的产品线与当下的市场结构"，因此是"最危险、最具颠覆性的行动路线"。

### `market-penetration` — 市场渗透

**Prompt** — 现有产品 × 现有市场。让现有客户买更多。风险：最低。杠杆：使用频次、复购、钱包份额、赢回、定价优化。

**Example** — 忠诚度计划提升复购

**Quality bar** — 现有产品 × 现有市场：让现有客户买更多。 风险最低的象限。 产品已被验证，客户关系存在，渠道运转——问题只是能从同一基础上榨出更多需求吗。

### `product-development` — 产品开发

**Prompt** — 新产品 × 现有市场。给已经懂的客户做新东西。风险：中等——买家关系已被验证，产品还没有。

**Example** — 现有渠道销售新产品线

**Quality bar** — 新产品 × 现有市场：给已经懂的客户做新东西。 中等风险——买家关系已被验证，产品没有。 最致命假设是你能否造出新产品而**不破坏**现有产品（共享成本基础、渠道、品牌、团队注意力）。

### `growth-rationale` — 增长理由

**Prompt** — 对所选象限写明：(a) 哪些维持不变；(b) BMC 哪些块必须改；(c) 每个候选最致命的假设；(d) 与现在其它事情并行推进的真实容量。

**Example** — 复用现有生产基础；渠道改为 D2C；风险：D2C 单位经济在放大时崩

**Quality bar** — 决策收口区。 候选放到各自象限并对比之后，团队在这里写下**承诺**——选了哪个象限、改什么、显式推迟什么。

## Colour legend

- `0` — **候选增长动作**: 正在考虑的具体增长押注——写明产品、市场和单位规模。
- `1` — **维持不变的部分**: 从今天的业务原封不动延续过来的 BMC 块、客户关系、能力或渠道。
- `2` — **必须改的部分**: 动作所需的新渠道、新资源、新伙伴或新成本结构。
- `3` — **最致命假设**: 如果不成立就会让动作崩盘的具体假设。可被证伪，带指标。
- `4` — **容量约束**: 可能迫使你顺序而非并行推进的组织容量 / 管理带宽限制。

## Pairs with

这张画布常和这些一起用,做完后引导用户接下去做:

- `business-model-canvas` — 所选的安索夫象限直接预言 BMC 哪些块必须改：渗透 → 渠道/客户关系；市场开发 → 客户细分/渠道；产品开发 → 价值主张/关键资源；多元化 → 几乎所有块。
- `portfolio-map` — 多元化动作应该成为 Portfolio Map 上新的 Explore pin；产品 / 市场开发应该是现有 pin 上的分支或移动。
- `design-criteria-canvas` — 编码下一个动作必须满足的约束（必须复用现有渠道、必须复用现有生产基础），防止团队无意中从一个象限滑到另一个。
- `bcg-growth-share-matrix` — BCG 给现有业务分类；Ansoff 为单一业务选增长方向。多业务：先 BCG 在 SBU 间分配，再对每个业务用 Ansoff 选方向。单业务：跳过 BCG，直接用 Ansoff。
- `three-horizons-map` — H1 通常是渗透；H2 常常是产品或市场开发；H3 常常是多元化或新市场颠覆。把 Ansoff 象限放进时间组合的位置。
