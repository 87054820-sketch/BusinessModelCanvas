# 证据评分卡

## When to use

证据评分卡帮助团队判断：已经做过的实验到底证明了什么。它适合放在访谈、落地页、Concierge 测试、试点、预售或销售沟通之后使用，因为这时团队通常已经有很多活动记录，但还没有清楚地回答下一步是加码、转向、继续验证还是终止。

新手先记住一个核心概念：观点还不是证据。证据只有连接了清晰假设、测试设计、可观察行为和决策，才真正有用。客户说“我喜欢”通常弱于客户改变工作流、投入时间、提供数据、签署意向、预付费用或真实使用。

## Blocks

`zoneId` 是 API 的稳定标识符,`pingarden canvas write` payload 必须用它,不要翻译。

### `critical-assumption` — 关键假设

**Prompt** — 这张卡正在评估哪一个最关键、最脆弱的假设？

**Example** — 目标客户愿意为自动化报告支付月费

**Quality bar** — 这个模块用于写清正在评估的脆弱判断。 关键假设不是所有未解问题，而是如果为假就会让项目失败的判断。

### `current-evidence` — 当前证据

**Prompt** — 目前有哪些真实证据？写实验结果、行为数据、客户原话，不写观点。

**Example** — 8 位受访者中 6 位描述了现有绕行方案

**Quality bar** — 这个模块只记录真实发生了什么。 证据可以是观察到的行为、使用数据、签署承诺、付款、工单、工作流截图、试点结果或客户原话。 解释放到后面的模块；这里要尽量贴近原始来源。 好答案会包含方法、样本、日期和结果。

### `evidence-strength` — 证据强度

**Prompt** — 证据是弱、中、强？为什么足以或不足以支撑下一步？

**Example** — 弱：只有访谈意向，没有行为

**Quality bar** — 证据强度回答：团队应该给这条证据多大权重。 *Testing Business Ideas* 会按照成本、时间和证据强度来区分实验。

### `learning-velocity` — 学习速度

**Prompt** — 团队多久获得一次可改变决策的新证据？最近学到了什么？

**Example** — 两周跑完一次实验

**Quality bar** — 学习速度指测试产生“能改变决策的学习”的频率。 它不是完成了多少实验。 一个团队可以做很多浅测试却几乎没有学习；另一个团队可能只做一次尖锐的预售，就足以改变路线图。 好答案会写出节奏、最近学到了什么、以及因此改变了什么。

### `risk-reduction` — 风险下降

**Prompt** — 需求性、可行性、可营性风险分别下降了多少？还有什么没被验证？

**Example** — 需求性下降；可营性仍弱

**Quality bar** — 风险下降说明哪一种不确定性变小了。 需求性风险问客户是否真的想要；可行性风险问团队是否能交付；可营性风险问经济账是否成立；适应性风险问模型能否承受监管、伙伴或市场变化。 好答案通常是不对称的：“需求性下降，可营性未变。

### `portfolio-decision` — 组合决策

**Prompt** — 基于证据，下一步应该继续、转向、补实验、转入、加码还是终止？

**Example** — 继续：进入 Concierge 测试

**Quality bar** — 这个模块把证据转成资源分配。 决策可以是继续、转向、补实验、转移学习、加码投入或终止。 关键是避免“创新剧场”：项目不能因为活动看起来热闹，就一直消耗时间。 好答案会写明决策、支撑它的证据，以及什么条件下重新讨论。

### `next-experiment` — 下一轮实验

**Prompt** — 下一轮最便宜、最能改变决策的实验是什么？通过/失败标准是什么？

**Example** — 2 周内预售给 5 个目标客户；≥2 单付款才进入产品化

**Quality bar** — 下一轮实验应该是最便宜、且足以改变决策的测试。 它要直接针对剩余关键风险，尽可能使用更强的证据类型，并在开始前写清通过/失败标准。 好答案包含假设、方法、样本、时间盒、证据类型和决策规则。

## Colour legend

- `0` — **证据**: 观察事实、实验结果、原话或使用数据。
- `1` — **学习**: 团队认知发生了什么变化。
- `2` — **决策**: 继续、转向、转入、投入或终止。
- `3` — **风险**: 剩余不确定性或弱证据。
