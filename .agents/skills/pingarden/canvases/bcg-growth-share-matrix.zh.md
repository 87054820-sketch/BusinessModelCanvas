# BCG 增长份额矩阵

## 何时使用

当用户有多个业务、产品、品牌或区域，并希望按市场增长率和相对市场份额分类时，使用 BCG 增长份额矩阵。

## 何时不用

- 如果市场份额未知且无法合理估算。
- 如果组合主要是早期创新押注：使用 `portfolio-map` 或 `three-horizons-map`。
- 如果主要问题是行业力量而不是组合配置：使用 `business-model-environment` 或 `porters-five-forces`。

## 填写顺序

1. 定义可比较单元。
2. 估算每个单元的市场增长率。
3. 估算相对最大竞争者或领导者的份额。
4. 放入问题、明星、瘦狗或现金牛象限。
5. 为增长/份额假设添加证据便签。
6. 决定投入、收割、选择、剥离或重新定位。
7. 重要单元在投入前展开为 BMC。

## 红旗信号

- 象限位置基于观点而非数据。
- 所有问题业务被平均投入。
- 现金牛被过度收割，经济性受损。
- 瘦狗业务没有明确战略理由却被保留。

## Blocks

`zoneId` 是 API 的稳定标识符,`pingarden canvas write` payload 必须用它,不要翻译。

### `question-marks` — 问题业务

**Prompt** — 高市场增长、低相对份额。哪些业务值得选择性投入，哪些应在消耗过多现金前放弃？

**Example** — 我们只是小挑战者的高速增长品类

**Quality bar** — 问题业务处在增长市场中，但公司的相对份额不足。 它们需要选择，而不是自动获得资金。 写清这个业务能成为明星业务的证据；否则应尽早筛掉。

### `stars` — 明星业务

**Prompt** — 高市场增长、高相对份额。哪些业务需要持续投入以防守领导地位，并成为未来现金来源？

**Example** — 快速增长市场中的品类领导者

**Quality bar** — 明星业务在高增长市场中拥有高相对份额，通常需要投入来防守领导地位并捕捉增长。 重大明星业务应展开为 `business-model-canvas`，并核对资金需求，不要默认它一定会变成现金牛。

### `dogs` — 瘦狗业务

**Prompt** — 低市场增长、低相对份额。哪些业务应剥离、关闭、重新定位，或仅因战略理由保留？

**Example** — 平缓市场中份额弱的遗留产品

**Quality bar** — 瘦狗业务位于低增长市场且相对份额较低。 它们不一定毫无价值，但必须有明确保留理由。 选择收割、剥离、关闭、重新定位，或写清楚保留的战略理由。

### `cash-cows` — 现金牛业务

**Prompt** — 低市场增长、高相对份额。哪些业务应高效防守并产生现金，用来资助未来增长押注？

**Example** — 高利润的成熟品类领导者

**Quality bar** — 现金牛业务处在增长较慢的市场中，但相对份额高。 它们为组合供血，但仍需要高效防守。 写清收割哪些现金、必须保护什么优势，以及这些现金资助哪些未来押注。

### `portfolio-actions` — 组合动作

**Prompt** — 把象限位置转化为行动：投入、收割、选择、剥离、重新定位，或重新验证市场增长/份额假设。

**Example** — 投入两个问题业务，收割一个现金牛，剥离一个瘦狗

**Quality bar** — 这个区域把象限标签转成管理动作。 只有标签、没有动作，组合不会改变。 每个动作都应写清负责人、证据来源、资金影响和复盘日期。

## Colour legend

- `0` — **业务 / 产品**: 待分类的可比较业务单元、产品线或品牌。
- `1` — **市场证据**: 关于市场增长、相对份额、竞争地位或品类成熟度的证据。
- `2` — **投入 / 建设**: 追加投入、扩张、强化或保护位置的动作。
- `3` — **收割 / 剥离**: 收割现金、减少投入、剥离或退出的动作。
- `4` — **重新定位问题**: 关于业务是否应迁移、聚焦、合并或重新定义的问题。

## Pairs with

这张画布常和这些一起用,做完后引导用户接下去做:

- `portfolio-map` — BCG 增长份额矩阵用于市场增长率/相对份额视角；当创新风险、颠覆风险和收益变化比市场份额更重要时，回到业务组合地图。
- `business-model-canvas` — 任何需要投入、收割、剥离或重新定位的业务，在决策前都应展开为 BMC。
- `business-model-environment` — 用商业模式环境信号证明市场增长假设，并挑战象限位置是否仍然成立。
