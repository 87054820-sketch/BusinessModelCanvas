# 战略画布(蓝海战略)

## 何时使用

战略画布(W. Chan Kim 和 Renée Mauborgne 的**蓝海战略**)用于**把竞争定位可视化** —— 把你的"价值曲线"和竞争对手的曲线在行业各竞争因素上画出来。目标:找出大家挤在一起的地方(红海),以及哪里可以用**形状明显不同**的曲线开出没人争的空间。

这张画布驱动的四个战略动作合起来叫 **ERRC**:**剔除(Eliminate)/ 减少(Reduce)/ 提升(Raise)/ 创造(Create)**。

## 何时不该用

- 内部产品路线图 —— 太粗了。战略画布是给行业级竞争定位用的。
- 没有竞争对手数据。整个画布的意义就是对比;只画自己一条曲线不是战略。
- 早期产品还没有竞争图景时 —— 先把行业图景测出来(桌面调研),再画。

## 填写顺序 —— 先看行业,再看自己,最后做 ERRC

这是图表型画布:X 轴列竞争因素(xAxisItems),Y 轴是投入水平(0 = 不参与,5 = 行业领先)。每个竞争对手(包括你)是一个 `pinClass`;每个 pin 把这个竞争者定位在某个 X 因素上的某个 Y 水平。同色 pin 连起来 = 一条价值曲线。

1. **列出竞争因素**(X 轴)。从客户**实际对比**的东西出发 —— 不是你希望他们对比的。看行业媒体、客户评论、赢单/丢单访谈。默认(价格 / 质量 / 服务 / 速度 / 品牌 / 创新)是起手套餐;换成行业特定的因素。
2. **先画在位者。** 两到三个最大的竞争对手。每个因素都打分。标好证据 —— 公开价格、NPS 报告、功能对比表。
3. **画破坏者 / 细分玩家。** 他们的曲线往往暗示行业要往哪走。
4. **诚实画你自己。** 不要乐观打分。如果你给自己服务打 4 但客户打 2,你就是 2。
5. **看不同的形状。** 一条能赚钱的蓝海曲线**几何形状明显不同** —— 在对手低谷处你高,在对手高峰处你低。如果你的形状和某个对手一样,你就在他的红海里。
6. **应用 ERRC** 起草你未来态的曲线:
   - **剔除**:哪些因素该**砍掉**(目前在玩但不必要)?
   - **减少**:哪些该**低于行业标准**(过度供给)?
   - **提升**:哪些该**高于行业标准**(供给不足)?
   - **创造**:哪些因素行业里**完全缺失**而你可以开辟?

## 跨模块一致性

- **每个对手的曲线必须画在同一组因素上。** 不要为某个对手新加他独有的因素 —— 那些归到 ERRC 的"创造"。
- **打分相对行业,不是绝对值。** 5 分 = "在这个因素上行业最好",不是"完美"。
- **你的未来态曲线要有不同的形状,不只是更高的分数。** 处处比所有对手分数高是幻想 + 烧钱 —— 通常意味着你在按他们的规则玩。
- 因素列表 5-10 个。少了过简化;多了曲线读不懂。

## 反模式 —— 不允许

- ❌ **处处比对手高的幻想。** 在每个因素上都赢的曲线。那不是战略,那是一份烧钱声明。
- ❌ **只列你独有的因素。** 那不在主图上 —— 归到 ERRC 的"创造"象限。
- ❌ **没有对手数据就画。** 你在猜他们的位置。先做桌面调研。
- ❌ **给自己乐观打分。** 用客户侧数据 —— 评论、问卷、流失。内部主观判断是错误来源。
- ❌ **把 ERRC 当头脑风暴。** ERRC 由曲线形状逼出来 —— 在画完之后做,不是之前。
- ❌ **跳过"剔除"动作。** 大多数团队觉得"提升 + 创造"很容易,从不"剔除"。但剔除省下来的钱才是创造的资金。

## 语气

每个 pin 的注释都要引来源("来自 G2 评分均值"、"来自行业刊物 X 的功能矩阵")。整张图的价值完全取决于打分的合法性;不透明的数字会让整个练习失效。

## Blocks

`zoneId` 是 API 的稳定标识符,`pingarden canvas write` payload 必须用它,不要翻译。

### `chart-area` — 图表区

**Prompt** — 图表区只放价值曲线的竞争因素、曲线类别和评分点；ERRC 要点、解释和结论写入 Story。

**Example** — 先画当前竞争者，再画未来曲线

**Quality bar** — 这个模块就是战略画布本身：横轴是竞争因子，纵轴是投入/表现水平，每条价值曲线代表一个竞争者或战略选项。 它不是 feature 表，而是一张视觉论证：这个行业今天在比什么，一个新战略如何改变价值基础。

## Colour legend

- `0` — **洞察**: 客户能感知的因子、非顾客洞察或行业假设。
- `1` — **ERIC 动作**: 围绕某个因子的剔除、减少、提升或创造理由。
- `2` — **证据**: 支撑曲线的客户行为或案例证据。
- `3` — **风险**: 新价值曲线带来的成本、采用或模仿风险。

## Pairs with

这张画布常和这些一起用,做完后引导用户接下去做:

- `business-model-canvas` — 把差异化后的价值曲线带回 BMC 的价值主张和客户细分 —— 整个模型通常都会跟着改。
- `value-proposition-canvas` — ERIC 动作(剔除 / 减少 / 提升 / 创造)直接对应 VPC 的痛点缓解和收益创造。
