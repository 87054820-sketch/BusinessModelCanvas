# 平台生态地图

## When to use

平台生态地图解释的是一种“价值来自多方交互”的业务，而不是一家公司把产品沿着管道推给客户。它适合市场平台、支付网络、应用商店、开发者生态、创作者平台、出行平台和 AI 生态。核心问题是：谁需要谁？为什么系统变大以后会更强，或者反而更难治理？

平台不是“有很多用户的产品”。它至少有两个参与方，并且有一个可重复的**核心交互**。一边创造或供给**价值单元**，另一边消费、购买、使用或回应它。网络效应指一边或多边参与者增加，会让同边或另一边参与更有价值。多归属指参与者可以同时使用多个平台，这会削弱锁定。

## Blocks

`zoneId` 是 API 的稳定标识符,`pingarden canvas write` payload 必须用它,不要翻译。

### `demand-side` — 需求侧

**Prompt** — 平台服务哪些需求侧参与方？他们为什么来？

**Example** — 乘客需要更快打到车

**Quality bar** — 需求侧是想要消费、购买、搜索、预订、雇佣、使用或回应价值单元的参与方。 不要把所有人都叫“用户”。 平台可能面对乘客、旅客、商户、广告主、开发者、创作者、患者或企业，每一类都有不同任务。 好答案会写清具体参与方、任务、动机和成功条件。

### `core-interaction` — 核心交互

**Prompt** — 平台最小可行的价值交换是什么？谁生产价值单元，谁消费？

**Example** — 房东发布房源，旅客搜索、预订、评价

**Quality bar** — 核心交互是让平台产生价值的最小、可重复交换。 平台架构通常包含参与方、价值单元，以及过滤或匹配机制。 如果你不能用一句话描述核心交互，这个平台很可能还只是功能堆叠。 好答案会说明谁生产价值单元、谁消费它，以及平台如何帮助匹配、过滤或交易。

### `supply-side` — 供给侧 / 互补方

**Prompt** — 供给侧或互补方是谁？他们为什么愿意参与？

**Example** — 司机用闲置时间接单

**Quality bar** — 供给侧创造价值单元，或让价值单元更有用。 在市场平台里，它可能是房东、司机、卖家或服务商；在生态系统里，它可能是开发者、硬件厂商、内容创作者、代理商或数据伙伴。 好答案会解释供给方为什么参与、贡献什么努力或资产、期待什么回报。

### `network-effects` — 网络效应

**Prompt** — 网络效应来自同边还是跨边？增长如何让下一位参与者更有价值？

**Example** — 更多司机缩短等待时间，吸引更多乘客

**Quality bar** — 网络效应解释参与者增加如何改变价值。 同边效应发生在同一边，例如更多开发者吸引更多开发者；跨边效应发生在不同边之间，例如更多司机缩短乘客等待时间。 负向效应也重要：拥堵、垃圾信息、低质量供给或供应者之间竞争过度，都可能降低价值。

### `governance-trust` — 治理与信任

**Prompt** — 平台用什么规则、审核、声誉或激励机制保证质量和信任？

**Example** — 双向评价

**Quality bar** — 治理是平台设置规则、处理争议、筛选质量并保护核心交互的方式。 信任不是一句口号，而是通过身份校验、声誉、评价、保障、准入规则、API 政策、激励和执行机制设计出来的。 好答案会把治理机制和具体失效模式连接起来。

### `monetization` — 变现机制

**Prompt** — 平台如何收费？收费是否会伤害核心交互或参与方激励？

**Example** — 交易抽佣

**Quality bar** — 平台变现设计的是谁付费、何时付、付多少、为什么付。 它同时也是治理，因为定价会改变激励。 平台可以通过交易抽佣、订阅、上架费、广告、增值工具、数据服务或 take rate 收费。 好答案会解释变现如何保护核心交互。

### `cold-start-risks` — 冷启动 / 风险

**Prompt** — 先激活哪一边？有哪些监管、欺诈、质量、补贴或反噬风险？

**Example** — 先补贴供给侧形成覆盖

**Quality bar** — 冷启动是鸡生蛋问题：需求等供给，供给等需求。 平台启动通常需要种子供给、补贴、借用既有网络、聚焦微市场、引入标杆参与者，或暂时用人工方式完成交互。 好答案会写清最难激活的一边、第一块市场边界、不可规模化的种子动作，以及可能击穿信任的风险。

## Colour legend

- `0` — **参与方**: 需求侧、供给侧、互补方或生态参与者。
- `1` — **交互**: 核心价值交换、匹配或交易流程。
- `2` — **治理**: 规则、信任机制、质量控制或激励。
- `3` — **风险**: 冷启动、欺诈、监管或变现风险。
