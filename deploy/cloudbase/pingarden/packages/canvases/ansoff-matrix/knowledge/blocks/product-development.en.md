# Product Development (产品开发)

New product × existing market: build new offerings for customers you already understand. Medium risk — the buyer relationship is proven, the product is not. The riskiest assumption is whether you can build the new product *without breaking* the existing product (shared cost base, channels, brand, team focus).

## Core moves

- **Adjacent SKU extension** — new product line that fits the existing brand and channel.
- **Up-market product** — premium offering for the current customer base at higher price point.
- **Service add-on** — new service module sold to existing product customers.
- **Platform extension** — add a new feature, integration, or workflow on the same platform.

## Core Questions

- Does the new product share the cost base, channels, brand, or team with the existing one? If not, this is closer to diversification.
- What is the cannibalisation risk — will the new product replace or add to the existing customer's purchase?
- Can the existing brand carry the new product, or does it need a new brand?
- What is the development cost and time-to-market relative to the predicted revenue?
- Do existing customers want this, or are you building because the team finds it interesting?

## Examples

- "Stitch Fix launching Men's Apparel — same algorithm + styling team + brand, new product. Same customer-base relationship extended."
- "Spotify launching Podcasts — same listening platform, new content type. Existing subscribers + advertisers, new content acquisition cost structure."
- "Enterprise SaaS adding a security module sold to existing customers — same buyer, new product, leveraged sales motion."

## How it flows into BMC

Product Development primarily changes BMC **Value Propositions** (new offering), **Key Activities** (development of new product), and **Key Resources** (capabilities for the new product). Customer Segments stays largely the same — if it shifts substantially, the move is market development or diversification.
