# 何时使用波特价值链

当团队有了战略意图（"拼质量"/"做最低成本"/"服务驱动"），但还说不清**哪些内部活动必须改变**才能兑现这个意图时使用。这张画布把公司拆成 9 类活动——5 类主要活动（进货物流 → 生产运营 → 出货物流 → 营销与销售 → 售后服务）+ 4 类支持活动（企业基础设施、人力资源管理、技术开发、采购），加上由此产生的利润。它定位竞争优势在哪一环形成，以及活动之间的联动在哪里互相加强或互相打架。

## 适合用它的时刻

- 战略意图已知（"做最低成本"/"拼质量"），但团队对该投资哪些能力还争不出结论。
- 利润在收窄，但没有任何单一活动的负责人能解释为什么——漏点通常在两类活动的**联动**处，而非单一活动内部。
- 团队准备填 BMC 的关键活动 / 关键资源 / 关键伙伴，需要先有更高分辨率的视图。
- 在位者在问：现有能力里哪些是真正的护城河、哪些只是大宗商品。
- 挑战者在问：必须在哪个活动上做到比对手更强，才能赢——而不是只是"做得好"。

## 不适合用它的时刻

- 问题是"我们做什么生意"——先用 **商业模式画布**，再回到价值链做执行细节。
- 问题是"行业结构怎样"——用 **波特五力**，它在公司之上，不在公司之内。
- 公司是平台 / 双边市场，承重活动是网络协调而非实体活动——以 **平台战略** + **平台生态地图** 作为主透镜。
- 问题是"组织真的能执行这套战略吗"——用 **麦肯锡 7S** 检查软硬要素一致性；价值链定位活动，7S 检查组织能不能跑这些活动。

## 与邻近框架的差异

- **BMC 的关键活动 / 关键资源 / 关键伙伴** 是同样 9 类活动的高度概括；价值链是高分辨率版本。两者并用：BMC 用作一页纸总览，价值链在"关键活动 = 生产运营"过于含糊时使用。
- **麦肯锡 7S** 问软硬要素是否一致。价值链问活动层级的优势在哪。一套依赖活动 X 但 7S 不匹配的战略必然失败。
- **平台战略** 是网络效应主导的多边业务的主透镜。价值链仍适用，但大部分权重转向技术开发 + 采购。

## 出自原著

> *"竞争优势不能仅从整体上看一家公司。它源自公司在设计、生产、营销、交付与支持产品过程中所执行的许多分立的活动。每一个活动都可能影响公司的相对成本地位，并构成差异化的基础。"*
>
> —— Michael E. Porter, *Competitive Advantage*, Free Press, 1985，第 33 页。

书中引入价值链作为诊断工具。Porter 的核心主张是："竞争优势"是一个掩盖了"哪些具体活动真正创造它"的聚合概念。这张画布继承了这种诊断态度：不要说"我们靠质量竞争"，而要说"生产运营 + 售后服务 + HRM 共同创造质量，采购予以加强"。

## 什么样算合格

- 每个活动写了 2-4 个具体子活动，不只是一个类别标签。
- 至少 2 条联动（紫色）可见——真正的差异化大多来自联动。
- 支持活动的密度与主要活动一致，没有被跳过。
- 成本优势 vs 差异化优势的颜色出现并被讨论过，没有以"以后再说"为由省略。
- 利润区写明了钱到底从哪来，附带粗略数字。
- 每个承载优势的活动都通过后续 sticky 映射到 BMC 的某个块上。
