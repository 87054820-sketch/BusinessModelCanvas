# Could Haves

These are useful extras that can wait. They may improve experience, add differentiation, or unlock future options, but they should not block current validation.

## How to write

- Include criteria that matter only if resources allow.
- Keep non-critical ideas out of Must and Should.
- Treat these as candidates for a later iteration roadmap.

## Examples

- “Could support multiple interface languages”
- “Could offer advanced analytics reports”
- “Could deeply integrate with third-party CRMs”
