# PESTEL 宏观环境分析

PESTEL 把宏观环境拆成 6 类长周期外部力量：**P** 政治（Political）、**E** 经济（Economic）、**S** 社会（Social）、**T** 技术（Technological）、**E** 环境（Environmental）、**L** 法律（Legal）。它是上游的环境扫描，产出的是「信号」而不是「决策」。在你决定打哪一场行业竞争之前，先看清世界正在变什么。

## 何时使用

PESTEL 适合用在战略迭代的起点：进入新市场、做周期性环境复盘、情景规划之前的准备阶段，或团队隐约觉得地基在动但还说不清是哪几股力。它也适合解决团队反复争论「未来」却没人能锚定的死循环——PESTEL 强制你回到可观察的外部力量，而不是各自的直觉。

不要把 PESTEL 当成「PPT 里提到过的事情」清单。每一格都必须装一个具体信号（一条监管、一个人口结构变化、一条技术曲线），并且能写出对应的商业模式 `so-what`。

## 与邻近框架的差异

- **商业模式环境扫描（Business Model Environment Scan）** 是 Strategyzer 的 4 力下游版本：直接拿 BMC 去顶趋势、市场、行业、宏观四股力。PESTEL 在宏观侧更宽更细，环境扫描更紧更贴 BMC。自然的接力是 **PESTEL 信号 → 环境扫描 4 力 → BMC 压力点**。具体的覆盖映射见下方专门一节。
- **波特五力** 与环境扫描同高度，但聚焦行业竞争经济学，不在宏观侧。先用 PESTEL 找到正在变化的东西，再用五力问利润压力来自哪里。
- **情景规划（Scenario Planning）** 消费 PESTEL 的输出：在高不确定性 + 高影响的 PESTEL 力量中挑出 2 个做情景矩阵的两个轴。

## 与商业模式环境扫描（BMEScan）的关系

PESTEL 和 BMEScan 不是相互替代的工具，而是「上游 → BMC」同一条管道上的不同高度。它们的差异是结构性的，值得仔细对照：

| 维度 | PESTEL | 商业模式环境扫描 |
| --- | --- | --- |
| 分桶数 | 6 类（P · E · S · T · E · L） | 4 力（趋势 · 市场 · 行业 · 宏观） |
| 高度 | 更宽的宏观环境 | 更紧、贴近 BMC |
| 输出 | 原始可观察信号 | BMC 的压力点 |
| 行业层 | 没有 | 明确的「行业力量」（竞争者 / 新进入者 / 替代品 / 供应商 / 利益相关方） |
| 客户层 | 隐含在 Social/Economic 里 | 明确的「市场力量」（细分 / 需求 / 转换成本 / 收入吸引力） |
| 何时启动 | BMC 还没成形时 | BMC 已存在、需要压力测试时 |

**覆盖映射** —— PESTEL → BMEScan：

- PESTEL **政治** + **法律** → BMEScan **宏观力量**（监管、公共基础设施）和部分**行业力量**（进入壁垒）。
- PESTEL **经济** → BMEScan **宏观力量**（资本市场、经济基础设施）和**市场力量**（收入吸引力、付费意愿）。
- PESTEL **社会** → BMEScan **关键趋势**（社会文化趋势）和**市场力量**（细分需求）。
- PESTEL **技术** → BMEScan **关键趋势**（技术趋势）和**行业力量**（替代品、技术驱动的新进入者）。
- PESTEL **环境** → BMEScan **关键趋势**（社会文化 / 监管趋势）和**宏观力量**（大宗商品、供应链外部性）。

PESTEL 不覆盖、BMEScan 覆盖的：**自家行业里的直接竞争对手和利益相关方**。所以只跑 PESTEL，你能看见宏观风向，但看不到自己擂台里的对手。只跑 BMEScan，你能看见近距离压力，但会漏掉长周期的宏观潮（例如人口结构倒挂、气候转型时间线）。

**推荐工作流**：

1. 战略迭代起点先跑 PESTEL，产出 20–40 条宏观信号。
2. 按「影响 × 不确定性」打分，留下排名靠前的 8–12 条。
3. 打开 BMEScan，把 BMC 锚在中央。把留下来的 PESTEL 信号分发到 BMEScan 的 4 个分区：趋势区吸收 PESTEL 的 S/T/E-环境 类上升信号；宏观区吸收 PESTEL 的 E-经济/P/L；市场和行业再补足买方 / 竞争对手层。
4. 把每条 BMEScan 的 sticky 翻译成「这压在 BMC X 块上」的 pin，让团队看见哪些块必须改。

跳过 PESTEL 直接做 BMEScan：在已知业务做周期性复盘时可以；跳过 BMEScan 只做 PESTEL：只有在 BMC 还不存在、还没什么可压测的时候才能接受。

## 与 PinGarden 画布的对应

- `business-model-environment` 是工作画布：每一类 PESTEL 力量变成趋势 / 市场 / 行业 / 宏观四个分区里的若干 sticky。
- `scenario-matrix` 把不确定性最高的两条 PESTEL 力量当成两个情景轴。
- `business-model-canvas` 是 PESTEL 信号最终的落点——每条信号都要落到 BMC 某个块的压力上。

## 工作流

1. 定义分析单元（行业 + 地理 + 时间窗）。
2. 给 6 类 PESTEL 力量分别列 3–5 条可观察信号（事实，不是形容词）。
3. 给每条信号标方向（上升 / 下降 / 波动）和时间窗（当下 / 1–3 年 / 3–10 年）。
4. 用「影响 × 不确定性」打分，高影响高不确定性的格子标记出来给情景规划。
5. 对每条高影响信号写一句 `so-what`，关联到 BMC 某个块。
6. 把没收口的 `so-what` 带入环境扫描、五力或情景矩阵作为下一步。

## 案例中需要注意什么

- `patagonia` — 环境（气候）和社会（供应链伦理）不是外部背景，而是商业模式的承重结构。
- `nickel-bank` — 政治/法律（法国 PSD2 银行牌照放开）+ 经济（金融排斥的底层人群）共同解锁了一个此前在法律和经济上都不成立的模式。
- `transsion-africa` — 社会（多 SIM 行为、深肤色拍照需求）、经济（50 美元以下价格带）、技术（先功能机再 Android）共同定义了一个完全不同于苹果的宏观背景。
- `ping-an-group` — 政治/法律（中国金融监管）和技术（移动端健康 / 金融一体化）共同让生态战略可行。

## 常见误用

- 列了一堆趋势但没翻译成 BMC 影响。
- 把 PESTEL 当一次性 PPT 而不是周期性扫描例程。
- 把宏观力量（PESTEL）和行业竞争（五力）混在同一高度。
- 过度偏爱技术维度（因为技术故事性强），忽略社会 / 环境 / 法律。
- 把不确定性当风险打折——PESTEL 的信号不是概率，是必须分情景应对的不确定性。
