# 创新指标

创新指标用于回答一个核心问题：探索型项目是否真的在降低不确定性，还是只是消耗预算、制造活动感。

## 核心思想

成熟业务可以用收入、利润、增长率和效率指标管理；探索型业务过早使用这些指标，往往会逼团队伪装确定性。创新指标更关注：

- 关键假设是否被明确写出；
- 实验是否产生了真实证据；
- 证据强度是否足以支持下一轮投入；
- 风险是否下降；
- 项目在 Portfolio Map 上是否应该继续、转向、转入或终止。

## 与画布的关系

- 用 `evidence-scorecard` 记录一个项目或机会的证据状态；
- 用 `experiment-canvas` 设计和复盘单个实验；
- 用 `portfolio-map` 看多个项目的风险/收益移动；
- 用 `innovation-culture-map` 检查组织是否奖励真实学习。

## 质量标准

好的创新指标不会鼓励团队做更多活动，而是帮助团队做更好的决策。每个指标都应该能改变下一步动作：继续投入、换假设、补实验、转入业务单元，或停止。
