# 波特价值链分析

波特价值链（Porter, 1985）把一家公司拆成 9 类内部活动，对每一类问：这项活动制造了成本优势、差异化优势，还是两者都有、还是都没有？5 类**主要活动**把产品从输入推到客户手中（进货物流 → 生产运营 → 出货物流 → 营销与销售 → 售后服务）。4 类**支持活动**支撑它们（企业基础设施、人力资源管理、技术开发、采购）。要点是停止把「公司」当成一块整体来谈，转而精确找出利润和差异化在哪一环产生——又在哪一环漏掉。

## 何时使用

当团队已经有了一个战略意图（「靠质量竞争」「做最低成本」），但还说不清哪些内部活动必须随之改变时使用。当能力清单看着没问题、利润却持续被侵蚀时，价值链是合适的透镜——漏点几乎都在两类活动的衔接处，而不在单一活动内部。

不要把它当一张通用流程图。鉴别性的问题永远是「每个活动对竞争优势的贡献是什么」，而不是「这个活动做什么」。

## 与邻近框架的差异

- **麦肯锡 7S** 问的是战略、结构、系统、共同价值观、风格、人员和技能是否对齐——「软硬要素对齐」的透镜。价值链聚焦在活动颗粒度上的**竞争优势在哪里形成**。7S 问「组织是否自洽」；价值链问「我们到底在哪里赢」。
- **平台战略** 是多边生态系统的透镜。价值链是内部活动的透镜。一个平台业务也有价值链——但它真正的承重差异化通常来自网络效应，而不是单点活动级的成本或差异化。
- **BMC 的 关键活动 / 关键资源 / 关键伙伴**：价值链就是这 3 个 BMC 块的高分辨率拆解。对应是直接的：BMC KA ≈ 当前在公司内部的主要活动；BMC KR ≈ 每个活动所需的资产 / 人；BMC KP ≈ 故意外部化的活动。

## 与 PinGarden 画布的对应

- `business-model-canvas` — 每个价值链活动翻译到 BMC 的 关键活动 / 关键资源 / 关键伙伴 中的某个块（这 3 个块加起来覆盖 9 类活动）。差异化活动也应该出现在价值主张的支撑里。
- `design-criteria-canvas` — 编码活动级规则（如「进货物流必须支持当日库存再平衡」「生产运营毛利不低于 30%」），下一轮商业模式迭代必须遵守。

## 工作流

1. 给分析单元画 9 个格子（单一业务，不是整个集团）。
2. 每个格子里写出具体的子活动（如生产运营里：排程、品控、包装）。
3. 给每个活动标：成本水平、差异化贡献、能力强度。
4. 高亮**联动**：两个活动彼此加强（如采购品质让生产稳定）或彼此打架（如营销承诺的服务速度超过售后能力）。
5. 挑出 2–3 个承重的差异化活动——能力投入应该集中在这里。
6. 把结论翻译成 BMC KA / KR / KP 的改动和设计规则。

## 案例中需要注意什么

- `patagonia` —— 进货物流（负责任采购）+ 生产运营（公平贸易认证工厂）+ 售后服务（Worn Wear 旧衣修复）形成一条互相加强的链。差异化不在任何单一活动，而在它们之间的**联动**和价值观驱动的人力资源底盘。砍掉任何一个，价值主张就崩。
- `cainiao` —— 整个业务就是一条以服务形态对外销售的物流价值链：进货（商家仓）→ 运营（分拨、报关）→ 出货（末端网络）就是产品本身。营销和服务活动反过来反馈数据，重新优化运营——典型的紧耦合链条。
- `gillette` —— 生产运营（精密刀片制造）+ 营销与销售（大额品牌投入）+ 采购（特种钢供应）合力守住「剃刀-刀片」模式。刀片的成本优势来自运营规模；价格防御来自营销。
- `nestle-portfolio` —— 在组合层级，多个 SBU 共享采购和企业基础设施（统一全球财务、统一研发），每个 SBU 跑自己的主要链。这是多业务公司里支持活动作为承重连接组织的典型说明。

## 常见误用

- 把 9 个格子画成通用流程图，没有做优势评估。
- 单独评估每个活动，漏掉联动——真正的差异化大多来自联动。
- 因为「无聊」就跳过支持活动（特别是 HRM 和企业基础设施）——它们通常是最难复制的护城河。
- 忘了服务或平台业务的「实体」进货 / 出货物流几乎消失，技术开发和容量采购成为主导。
- 把价值链结论留在 PPT 上，没有翻译成 BMC 的 KA/KR/KP 改动——分析没改变模型就等于没做。

## 来源

- Michael E. Porter, *Competitive Advantage: Creating and Sustaining Superior Performance*, Free Press, 1985。
