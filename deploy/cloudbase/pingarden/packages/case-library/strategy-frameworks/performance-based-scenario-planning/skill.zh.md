# 基于绩效的情景规划

## 何时选择

- 用户需要运行一个情景规划项目，而不只是画几个情景。
- 赞助者支持、利益相关者参与、实施或评估本身就是风险。
- 用户问情景规划如何影响决策和组织学习。

## 填写 / 引导顺序

1. 明确目的、赞助者、范围、参与者和成功标准。
2. 用 `business-model-environment` 收集信号和不确定性候选。
3. 用 `scenario-matrix` 构造情景和稳健动作。
4. 用情景对 BMC、组合押注和设计准则做风洞测试。
5. 决定现在改变什么、保留什么选项、监测什么信号。
6. 评估结果：决策影响、学习、感知变化和后续动作。

## 红旗信号

- 目的还没清楚就安排工作坊。
- 情景写完但从未被使用。
- 没有评估计划。
- 利益相关者只是被告知，并没有参与。
