# 基于绩效的情景规划

基于绩效的情景规划是情景规划的组织执行层。它不替代情景矩阵，而是解释组织如何准备、运行、部署和评估一个情景项目，使情景真正影响决策、心智模型和绩效。

## 何时使用

当用户负责的是一个情景规划项目，而不只是一次情景工作坊时使用。它特别适合那些成败取决于领导支持、利益相关者参与、项目范围、实施和评估的场景。

如果用户只需要快速做一个 2×2 情景练习，不必使用它。核心多未来逻辑使用 `scenario-planning` 和 `scenario-matrix`；当组织采纳和决策影响很重要时，再使用这个框架。

## 五阶段流程

1. **项目准备**：明确目的、赞助者支持、范围、参与者、议题焦点和成功标准。
2. **情景探索**：收集信号、趋势、利益相关者视角、驱动力和不确定性候选。
3. **情景开发**：构造内部一致、能够挑战假设和心智模型的情景。
4. **情景实施**：把情景用于战略对话、风洞测试、BMC 压力测试、组合选择和行动设计。
5. **项目评估**：记录结果、学习、感知变化、决策影响和后续监测信号。

## 画布映射

- `business-model-environment` 提供信号和外部力量。
- `scenario-matrix` 生成情景故事和稳健动作。
- `business-model-canvas` 在每个未来中被压力测试。
- `portfolio-map` 比较哪些押注能跨未来成立。
- `design-criteria-canvas` 沉淀稳健动作和决策规则。

## 和 Scenario Planning 的区别

`scenario-planning` 是跨多个未来思考的总方法。`performance-based-scenario-planning` 是让情景工作在组织中被使用的项目系统。

## 常见误用

- 尚未定义目的和赞助者，就直接开始情景工作坊。
- 把情景开发当成项目终点。
- 没有把情景部署到战略决策或组合选择。
- 忽略感知、偏见和心智模型。
- 跳过评估，导致组织无法知道项目是否有帮助。
