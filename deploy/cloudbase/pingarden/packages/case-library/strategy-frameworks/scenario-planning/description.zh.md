# 情景规划

情景规划把商业模式环境扫描向前推进一步：不是只列外部机会和威胁，而是围绕驱动力和关键不确定性构造多个可信未来，并比较战略选择在不同未来下是否仍然成立。

## 核心思想

单一预测容易让团队过早押注一个未来。情景规划要求团队先识别驱动力，再找出真正会改变战略选择的不确定性，组合成若干个足够不同、但都合理的未来。目标不是预测更准，而是提升战略思考质量。

## 何时使用

当商业模式、组合或战略选择受到技术、监管、客户行为、行业结构、宏观环境或社会变化的高不确定性影响时使用。

如果问题只是运营排期、简单预测或低不确定性决策，不要使用它。也不要写“乐观/中性/悲观”三档版本，那只是把单一预测包装成情景。

## 与画布的关系

- 用 `business-model-environment` 收集趋势、市场、行业和宏观信号。
- 用 `scenario-matrix` 把关键不确定性转成多个未来。
- 用 `business-model-canvas` 检查当前或拟议商业模式受到的压力。
- 用 `portfolio-map` 比较哪些押注能跨多个未来成立。
- 用 `design-criteria-canvas` 把稳健动作转成设计标准。

## 实操流程

1. 定义焦点议题和时间范围。
2. 通过环境扫描收集外部信号。
3. 区分相对确定的趋势和真正的关键不确定性。
4. 选择最会改变战略选择的不确定性。
5. 构造几个内部一致的情景。
6. 用每个情景对 BMC、组合押注和设计准则做风洞测试。
7. 找出稳健动作、条件动作、需要保留的选项和早期监测信号。
8. 如果情景规划需要影响组织决策，就用 `performance-based-scenario-planning` 的项目方式来准备、实施和评估。

## 示例中看什么

`patagonia` 是当前高置信示例，因为该案例包含情景矩阵工作，并展示外部环境、市场和社会力量如何压力测试商业模式和战略姿态。

## 质量标准

好的情景不是“乐观/中性/悲观”三档预测，而是能迫使团队看到不同战略逻辑的多个世界。结论应该落到：保留哪些选项、监测哪些信号、哪些动作跨未来稳健、哪些决策会随情景变化。

## 常见误用

- 写出生动未来，却不改变任何决策。
- 把情景当成带概率的预测。
- 选择并不独立的不确定性。
- 忽略弱信号和早期预警指标。
- 停留在情景故事，没有用它们压力测试 BMC、组合和设计准则。
