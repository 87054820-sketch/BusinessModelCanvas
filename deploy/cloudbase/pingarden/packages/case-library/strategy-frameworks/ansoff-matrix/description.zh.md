# 安索夫矩阵

安索夫矩阵（Ansoff, 1957）把增长动作放在两个轴上——**现有产品 vs. 新产品** × **现有市场 vs. 新市场**——得出 4 条路径：

1. **市场渗透（Market penetration）** —— 现有产品 + 现有市场：让现有客户买更多。
2. **市场开发（Market development）** —— 现有产品 + 新市场：把同一个产品带到新地域、新人群或新使用场合。
3. **产品开发（Product development）** —— 新产品 + 现有市场：给已经懂的客户做新东西。
4. **多元化（Diversification）** —— 新产品 + 新市场：风险最高的象限，通常只有在核心业务成熟、新业务结构相邻，或客户 / 产品至少共享一层底盘时才成立。

4 个象限按执行风险排序，多元化承担最多失败模式。

## 何时使用

当一个业务面前有多条可能的增长路径，团队必须先点名「我们要的是哪一条」时，使用安索夫矩阵。它也常用来抵抗「无意中变成多元化」——团队常常一个功能一个功能地从渗透滑向多元化，却没人意识到性质变了。

不要用安索夫管理多业务组合（那是 BCG / 三层增长 / 商业模式组合管理的事）。它活在业务单元层级，问的是「这一个业务接下来怎么增长」。

## 与邻近框架的差异

- **BCG 增长份额矩阵** 用份额和增长给**现有业务**分类，决定投入 / 收割 / 剥离。安索夫问的是更前一步：当我们选择增长，往**哪个方向**？详细对照见下一节。
- **麦肯锡三层增长** 是时间组合层（今日核心 / 扩张中 / 未来选项）。安索夫是任何 Horizon 内的单业务动作——H1 通常是渗透；H2 常常是产品开发或市场开发；H3 常常是多元化或新市场颠覆。
- **商业模式组合管理** 统一 Explore/Exploit。安索夫指出下一个增长动作长什么样；组合管理决定它在业务组合地图上放在哪里。

## 安索夫 vs. BCG 矩阵：使用差异

两者都是关于增长的 2×2 矩阵，但回答的是**相反**的问题，且在不同层级工作。混用是战略 PPT 里最常见的矩阵错误。

| 维度 | 安索夫矩阵 | BCG 增长份额矩阵 |
| --- | --- | --- |
| 轴 | 现有/新**产品** × 现有/新**市场** | **市场增长率** × **相对市场份额** |
| 分析单元 | 单一业务 / 一张 BMC | 多业务组合（每个 SBU = 一个气泡） |
| 回答的问题 | 「这个业务接下来往**哪个方向**增长？」 | 「我们已经拥有的业务应该怎么处理？」 |
| 时间朝向 | **向前** —— 关于未来动作 | **快照** —— 关于当前组合位置 |
| 输出 | 一个方向选择（4 个按风险排序的象限） | 一个资源分配决策（投入 / 收割 / 剥离） |
| 风险轴 | 象限显式按执行风险排序（渗透 → 市场开发 → 产品开发 → 多元化） | 象限不按风险排序；每个象限有自己的风险结构 |
| 它**不**做什么 | 不告诉你哪些现有业务该投钱 | 不告诉你单个业务该往哪个方向扩张 |

**逐象限交叉映射** —— 给定一个现有业务的 BCG 分类，对应的安索夫下一步通常是什么？

- BCG **现金牛**（低增长 + 高份额）→ 安索夫**市场渗透**（守份额、提利用率）或受控收割。很少从牛业务本身做多元化。
- BCG **明星**（高增长 + 高份额）→ 安索夫**市场开发**（地理 / 细分扩张）或**产品开发**（产品延展），在份额还守得住时维持增长。
- BCG **问题**（高增长 + 低份额）→ 安索夫是决策工具：**加大渗透**抢份额、**产品开发**做差异化，或承认赢不了直接砍掉。
- BCG **瘦狗**（低增长 + 低份额）→ 安索夫几乎不在这里选渗透。要么原地收割，要么试**多元化**走出来——但仅在能讲清相邻性时。

**多业务组合时的推荐工作流**：

1. 先在组合层跑 BCG —— 把每个 SBU 分到 4 格之一。
2. 按格子决定：守哪些牛、投哪些星、押哪些问题、收哪些狗。
3. 对保留的业务（尤其是星和被押的问题）跑安索夫，决定增长**方向**：渗透、市场开发、产品开发还是多元化。
4. 把安索夫的输出带入 BMC + 业务组合地图执行。

**单业务时的推荐工作流**：跳过 BCG（只有一个气泡，矩阵没用），直接用安索夫选增长方向。

**常见混淆**：团队有时把安索夫的「多元化」象限当成组合决策，伸手去拿 BCG。这是错的——安索夫的多元化仍然是**单一业务**要扩张的方向；一旦你有了多个业务，BCG 才是合适的组合视角，安索夫则在每个业务上再用一次。

## 与 PinGarden 画布的对应

- `business-model-canvas` — 给定一个现有业务，所选的安索夫象限直接预言哪些 BMC 块必须改（渗透：渠道 / 客户关系；市场开发：客户细分 / 渠道；产品开发：价值主张 / 关键资源；多元化：几乎所有块）。
- `portfolio-map` — 多元化动作应该成为新的 Explore pin；产品 / 市场开发应该是在现有 pin 上的分支或移动。
- `design-criteria-canvas` — 编码下一个动作必须满足的约束（例如「必须复用现有渠道」「必须复用现有生产基础」），让团队不至于滑到错的象限。

## 工作流

1. 定义单元（一个业务 / 一张 BMC）。
2. 标出它目前的产品 × 市场位置。
3. 头脑风暴 2–4 个候选增长动作，每个放进一个象限。
4. 对每个候选写出 (a) 哪些维持不变；(b) 哪些必须改；(c) 最致命的假设。
5. 在期望收益 × 风险 × 战略契合度上对比候选。
6. 给每个增长周期选一个象限；不要在同一时期把两个象限叠加在同一业务上，除非显式规划了容量。

## 案例中需要注意什么

- `spotify` — 最早是**市场渗透**（瑞典音乐流媒体）；通过**市场开发**完成地理扩张；接着**产品开发**（播客、有声书）；现在试验**多元化**（线下演出、AI 功能）。4 个象限是被顺序重访的，不是只挑一次。
- `stitch-fix` — 在渗透之上叠**产品开发**的典型例子：女装 → 男装 → 童装 → 家居用品，每条新线复用同一套算法造型核心。
- `nintendo-wii` — 教科书级**多元化**：新产品（体感主机）+ 新市场（休闲非玩家 / 家庭），两个轴同时动。需要新伙伴、新渠道、新定价逻辑。
- `nestle-portfolio` — 在组合内并行多个安索夫动作：咖啡品类做渗透、新兴市场做市场开发、健康科学做产品开发、并购做多元化。有用的反例：在组合层级看，安索夫是单业务视角，必须先逐业务做完再聚合。

## 常见误用

- 把一个小功能增加叫「多元化」——大多数其实是产品开发。
- 把安索夫当一次性象限选择，而不是周期性决策。
- 因为其它象限「太无聊」就选多元化，却没说清新旧之间共享了什么。
- 把市场开发（同一产品、新买家）和产品开发（新产品、同一买家）混在一起——最致命的假设完全不同。
- 跳过「BMC 哪些块必须改」的映射，结果团队规划了增长却没有配套资源 / 伙伴 / 渠道。
