# 《Testing Business Ideas》阅读地图

这本书是 `experiment-canvas` 和 `evidence-scorecard` 的主要操作来源。它适合在团队已经有 BMC、VPC 或新业务想法之后阅读，用来把“我们相信”变成“我们准备如何验证”。

## 这本书解决什么问题

- 团队有很多想法，但不知道哪一个风险最大、应该先验证什么。
- 实验被做成营销活动或调研动作，没有明确假设和通过/失败标准。
- 证据强度混乱：把访谈意愿、点击、预售和真实使用都当成同等级证据。
- 实验结果出来后，团队不知道应该继续、转向、重做还是停止。

## 目录/阅读地图

1. **Design the experiment / 设计实验**：先写清假设、风险类型、实验对象、证据标准和决策规则。
2. **Test desirability / 验证需求性**：客户是否真的有任务、痛点、兴趣和购买动机。
3. **Test feasibility / 验证可行性**：团队是否有能力交付方案，技术、运营、供应和合规是否可行。
4. **Test viability / 验证可营性**：收入、成本、渠道、单位经济和增长逻辑是否成立。
5. **Measure evidence / 衡量证据**：按成本、时间和证据强度选择实验，而不是只选最容易做的。
6. **Decide next action / 决策下一步**：把实验结论连接到继续、转向、追加实验、缩小范围或停止。

## 和 PinGarden 画布怎么对应

- `experiment-canvas`：核心入口，用来定义假设、实验、指标和决策。
- `value-proposition-canvas`：提供需求性假设，如客户任务、痛点和收益是否真实。
- `business-model-canvas`：提供可营性假设，如渠道、收入、成本和关键伙伴是否成立。
- `evidence-scorecard`：记录证据强度和风险降低程度，避免实验结果散落在文档里。

## 建议重点读哪些实验

- **早期方向不清**：客户访谈、线上问卷、讨论区观察、搜索趋势分析。
- **价值主张不确定**：故事板、可点击原型、登陆页、烟雾测试。
- **交付能力不确定**：礼宾式实验、绿野仙踪、手工服务、单功能原型。
- **付费意愿不确定**：意向书、预售、众筹、价格测试。

## 常见误读

- **误读 1：做了实验就等于验证。** 没有预先定义通过/失败标准的实验，只是收集信息。
- **误读 2：用户喜欢就是强证据。** 真实购买、重复使用和付费承诺通常比口头反馈强。
- **误读 3：实验越大越可靠。** 早期目标是快速降低最大风险，不是做一个小型完整产品。
