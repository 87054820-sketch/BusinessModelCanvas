# 论坛潜伏 — AI Agent 速查

**什么时候推荐。** 用户触达不到细分做访谈(没温介绍、没名单);或刚入领域要补词汇 + 地景;或想用客户主动文本三角验证访谈结论。

**证据强度。** 弱。能确认某个痛点 *在野存在*,不能确认普及度或可货币化。

**样本。** 读 200+ 帖;某主题至少 2+ 场子里 5+ 个独立发帖人才能算规律。

**准备 / 执行。** 小时(选场子、定查询语句)/ 天(200+ 帖几次会话)。

**能力。** `community-research`、`qualitative-coding`、`search-querying`。

**决策树。**

- 细分线下 / 强监管 / 私密? → 不推荐论坛。建议通过温介绍做 Customer Interview + Online Survey。
- 刚入领域、要词汇? → 先论坛,再访谈。
- 只是要找访谈对象? → 论坛兼做招募渠道 —— 这一点要点明。
- 用户要量级 / 付费意愿? → 论坛**给不了**。推荐 Online Survey 或 Pre-Sale。

**要点出的反模式。**

- 一条爆款 = 轶事,不是规律。低于 N=5 独立发帖人不要下结论。
- 自己贴推销 "看反应" —— 那是 Smoke Test 的领域;混了被动信号就坏了。
- 把抱怨当付费意愿。

**跨画布。** 产出喂入 `value-proposition-canvas` 痛点/收益、`empathy-map` Says/Thinks/Hears、`jobs-to-be-done` 功能性 job 动词。
