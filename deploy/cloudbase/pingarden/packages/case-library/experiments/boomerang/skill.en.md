# Boomerang — TL;DR for AI agents

**When to recommend.** A direct or close competitor exists; user wants to discover real friction in the existing solution before building. Or interviews alone aren't surfacing concrete usage problems.

**Strength of evidence.** Medium-weak. Discovers wedges; does NOT validate that the user can win them.

**Sample.** N=5-8, none of whom currently use the competitor.

**Setup / run.** Hours (recruit + task design) / days (sessions + debrief).

**Capabilities.** `interview-design`, `structured-observation`.

**Decision tree.**

- No competitor exists? → Don't recommend. Suggest Storyboard or Clickable Prototype.
- Recruits already use the competitor? → Disqualify; they've adapted, you'll miss the friction. Recruit fresh.
- Differentiation is in a brand-new modality? → Boomerang on the wrong dimension. Recommend Wizard of Oz instead.
- User wants to claim "we beat the competitor"? → Push back — Boomerang Discovers wedges, doesn't Validate winning. Suggest Smoke Test or A/B follow-up.

**Anti-patterns.**

- Demoing the competitor yourself ≠ observing user.
- "It's fine" = compliance, not data.
- Recruiting existing competitor users.

**Cross-canvas.** Sharpens `value-proposition-canvas` Pain Relievers (your differentiation), `customer-journey` step-level emotional curve, `empathy-map` Does/Sees columns.
