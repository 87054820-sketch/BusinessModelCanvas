# 搜索趋势分析 (Search Trend Analysis)

用 Google Trends、Ahrefs / SEMrush 的关键词量、百度指数、Reddit / HN 搜索看:你在访谈里听到的"未被满足的需求"语言,是否在公开网络上有规模 —— 以及它在涨、在平、还是在跌。

## 什么时候用

- 访谈已经浮现具体问题语言("PR review 疲劳"、"AI 幻觉成本");想看这个语言是否走得远。
- 在投入工程之前给细分定大小。
- 对 Business Model Environment 的 "关键趋势" 那块做现实检验 —— 你假设的宏观趋势是不是真的在发生?
- 在两个候选细分中按搜索量挑一个。

不要用:

- 大部分 B2B 细分 —— 买家走分析师报告 + 口碑,不走 Google,搜索信号本来就弱。
- 还没有词汇的新问题框 —— 你会看到 "无量",但可能只是语言还没成形。

## 怎么跑

1. **把访谈里的语言翻译成 5-15 个查询候选。** 头部词("AI code review")和长尾("一天 80 个 PR 怎么过")都要。
2. **拉量级 + 趋势线。** 趋势用 5 年窗口,噪音用 12 个月。
3. **比候选之间的相对量级。** 10 倍差距是信号;同一噪音带里的 2 倍差距不是。
4. **看相关查询。** Trends 的 "上升" 面板经常出现你没想到的词汇。
5. **和 Reddit / HN 搜索数交叉验证。** 量高但社区不讨论,关键词可能被某个既有方案品类吃掉了。

## 什么算做得好

- 某个头部词明显超过备选 3-10 倍。
- 趋势线 5 年内持平或上扬(下降趋势是致命 —— 不要逆重力做)。
- 词汇能对得上访谈的语言。
- 地域和季节性能对上目标细分。

## 反模式

- ❌ 只看一个头部词。要 5-15 个查询才看出图景。
- ❌ 把绝对量级当成可触达市场 —— 高搜索量也可能是爱好者。
- ❌ 把 30 天的小高峰当趋势。要看 5 年。
- ❌ 跳过 Reddit / HN 交叉 —— Google 量可能被针对这个词的 SEO 垃圾内容拉高。
