# 平安集团组合转型

## 背景与张力

平安不是一个单一的数字医疗故事。它的起点是强大的金融服务核心：保险、银行、财富管理、代理人网络、风险模型、资本和信任。这些核心业务带来收益，但也让集团暴露在传统金融机构的限制中。如果增长只依赖向同一批客户销售更多金融产品，平安会受到科技平台、监管变化和用户数字化习惯的持续压力。

组合颗粒度应固定为“可独立运营的平台 / 业务单元”：保险核心、银行与财富、平安好医生、车主生态、金融壹账通等。

## 动态移动

| 组合对象 | 起点 | 移动 | 组合动作 | 判断标准 |
|---|---|---|---|---|
| 保险核心 | Exploit 高收益区 | 持续开发与保护 | Improve / Protect | 保持收益，同时降低平台脱媒和监管风险 |
| 银行与财富 | Exploit 稳定贡献区 | 与生态交叉销售 | Improve | 是否提高客户生命周期价值 |
| 平安好医生 | Explore 医疗平台选项 | 证据足够才可转入规模化 | Invest / Persevere / Transfer | 用户频率、医疗供给、支付方和保险协同是否成立 |
| 车主生态 | Explore 相邻场景 | 用车主服务提高触点频率 | Pivot / Persevere | 是否能降低获客成本或提高保险互动 |
| 金融壹账通 | Explore 到 Transfer 候选 | 内部能力外部化 | Spinout / Transfer | 企业客户付费和交付成本是否成立 |

## 机制

机制是组合邻近性。平安可以把核心金融业务中的客户信任、理赔数据、资本和分销能力，用来探索医疗、车主和企业金融科技平台。同时，这些平台也可能反哺核心业务：提高客户互动频率、降低服务成本，或形成新的数据循环。

这就是为什么单张 BMC 不够。pin 与 pin 之间的关系才是重点：保险业务为探索提供资金，数字平台创造更频繁触点，企业金融科技把内部能力变成外部平台。

## 文化与风险

这种组合需要支持创新的文化。如果每个平台都用成熟保险业务的 KPI 衡量，Explore 项目会过早死亡。如果每个探索项目都被无限保护，集团又会浪费资本。文化必须支持低成本实验、证据评审、转入决策和有纪律地终止项目。

## 迁移启发

对成熟公司来说，组合管理不是转型口号，而是一套治理系统：定义组合颗粒度，区分 Explore 与 Exploit，用风险和收益定位每个对象，再随着时间选择动作。
